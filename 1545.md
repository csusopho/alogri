[Problem: 1545. 找出第 N 个二进制字符串中的第 K 位](https://leetcode.cn/problems/find-kth-bit-in-nth-binary-string/description/)

### 方法：暴力 & 分治

注意，题目先 $invert$ 再 $reverse$，比如串 $011$ 先变 $100$ 再变为 $001$。

本题限制 $n\leq 20$，可以直接 **模拟** 字符串生成过程。

暴力解法可行：一个 $for$ 循环，从 $S_1$ 一直迭代生成到 $S_n$。生成完毕后，直接返回索引位置为 $k - 1$ 的字符。

代码如下，已附加注释：

```Python
# python
class Solution:
    def findKthBit(self, n: int, k: int) -> str:
        # 初始条件 S1
        s = "0"
        
        # 逐步生成
        for _ in range(2, n + 1):
            # 先反转
            reversed_s = s[::-1]
            # 再按位翻转
            rev_inv_s = "".join('1' if char == '0' else '0' for char in reversed_s)
            # 拼接
            s = s + "1" + rev_inv_s
            
        # 返回
        return s[k - 1]
```

```Java
// java
class Solution {
    public char findKthBit(int n, int k) {
        // 初始状态 S1
        StringBuilder sb = new StringBuilder("0");
        
        // 逐步生成
        for (int i = 2; i <= n; i++) {
            // 当前左半部分 Si-1 的长度
            int len = sb.length();
            // 拼接中间的 1
            sb.append('1');
            
            // 倒序遍历前半段，把翻转后的字符追加到末尾
            for (int j = len - 1; j >= 0; j--) {
                char c = sb.charAt(j);
                sb.append(c == '0' ? '1' : '0');
            }
        }
        
        // 返回
        return sb.charAt(k - 1);
    }
}
```

- 时间复杂度： $O(2^n)$，每一次循环字符串长度翻倍，生成 $S_n$ 需要处理约 $2^n$ 个字符
- 空间复杂度： $O(2^n)$，生成的结果均保留在变量中

### 分治

实际上，不需要真正生成整个字符串。寻找规律：

- $S_1 = \text{0}$，长度为 $1$，也即 $2^1 - 1$
- $S_2 = \text{011}$，长度为 $3$，也即 $2^2 - 1$
- $S_3 = \text{0111001}$，长度为 $7$，也即 $2^3 - 1$
- 由此可以得出，串 $S_n$ 的长度恒定为 $2^n - 1$

每次拼接的中间字符 $1$，其所在的位置正好是整个字符串的最中间，即第 $2^{n-1}$ 个字符。

在一个长度为 $2^n - 1$ 的字符串 $S_n$ 中找第 $k$ 个字符，中点位置是 $mid = 2^{n-1}$。把 $S_n$ 分为三部分：左半边 $S_{n-1}$ + 中间的 $1$ + 右半边 $S_{n-1}$ 的反转且取反。

- **情况 A：$k = mid$**。此时，正好命中了最中间的拼接字符，该字符固定为 $1$。
- **情况 B：$k < mid$**，说明目标字符在左半部分，而左半部分就是 $S_{n-1}$，所以问题转化为：**在 $S_{n-1}$ 中找第 $k$ 个字符**。
- **情况 C：$k > mid$**，说明字符在右半部分。右半部分是 $S_{n-1}$ 经过 $invert$ 和 $reverse$ 后得到的。需要推导出它在原本的 $S_{n-1}$ 中对应的是第几个字符：
  - 右半部分相对于中心的偏移量是 $x = k - mid$。
  - 因为有 $reverse$，右边正数第 $x$ 个字符，其实是原本 $S_{n-1}$ 的 **倒数** 第 $x$ 个字符。
  - $S_{n-1}$ 的总长度是 $mid - 1$。它的倒数第 $x$ 个字符正向索引是： $(mid - 1) - x + 1 = mid - x$。
  - 代入 $x = k - mid$，对应索引为： $mid - (k - mid) = 2 \times mid - k = 2^n - k$。
  - 同时，因为有 $invert$ 操作，这个字符还需要被 **翻转一次**。

**总结逻辑**：只需要用一个变量 $cnt$ 记录一共进行了多少次翻转，即落入右半部分的次数。不断缩小 $n$ 的范围，直到 $k = mid$ 或者 $k = 1$。最后根据 $cnt$ 的奇偶性返回真正的字符。

代码如下，已附加注释：

```Python
class Solution:
    def findKthBit(self, n: int, k: int) -> str:
        # 记录被翻转的次数
        cnt = 0
        
        while k > 1:
            # 当前 Sn 的中间位置
            mid = 1 << (n - 1)
            
            if k == mid:
                # 命中了正中间的 1
                return "1" if cnt % 2 == 0 else "0"
            elif k > mid:
                # 落在了右半部分，需要将其映射回左边 S_{n-1} 的对称位置
                k = (1 << n) - k
                cnt += 1
            
            # 变小为 S_{n-1}
            n -= 1
        
        # 回到了最基本的 S1
        return "0" if cnt % 2 == 0 else "1"
```

```Java
// java
class Solution {
    public char findKthBit(int n, int k) {
        // 记录被翻转的次数
        int cnt = 0;
        
        while (k > 1) {
            // 当前 Sn 的中间位置
            int mid = 1 << (n - 1);
            
            if (k == mid) {
                // 命中了正中间的 1
                return cnt % 2 == 0 ? '1' : '0';
            } else if (k > mid) {
                // 落在了右半部分，需要将其映射回左边 S_{n-1} 的对称位置
                k = (1 << n) - k;
                cnt += 1;
            }
            
            // 变小为 S_{n-1}
            n -= 1;
        }
        
        // 回到了最基本的 S1
        return cnt % 2 == 0 ? '0' : '1';
    }
}
```

- 时间复杂度： $O(n)$，最多循环 $n$ 次
- 空间复杂度： $O(1)$，仅使用常数个额外变量
