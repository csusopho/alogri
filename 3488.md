[Problem: 3488. 距离最小相等元素查询](https://leetcode.cn/problems/closest-equal-element-queries/description/)

### 方法：模拟

一句话概述：**环形数组中**，指定下标元素，找到和它值相同的其他元素的 **最小距离**。

在长度为 $n$ 的数组中，下标 $i$ 和下标 $j$ 之间的距离有两种走法：

1. 直接距离： $|i - j|$
2. 绕过边界的距离： $n - |i - j|$

因此，环形数组中的 **最短距离** 即为： $\min(|i - j|, n - |i - j|)$。

对于数组中某个特定的值（例如数字 $x$），假设它出现在下标 $[p_0, p_1, p_2, ..., p_{m-1}]$。对于下标 $p_k$ 处的元素，要找到与它最近的相同元素，我们只需要检查它的 **前一个** 相同元素和 **后一个** 相同元素（在环形意义下）：

- 左边的邻居是 $p_{k-1}$（若 $k=0$，则左邻居是 $p_{m-1}$）。
- 右边的邻居是 $p_{k+1}$（若 $k=m-1$，则右邻居是 $p_{0}$）。

---

解题一共三步：

**分组记录下标**：遍历数组 $nums$，用一个哈希表 $Map$ 记录每个数值出现的所有下标。

**预计算每个下标的结果**：

- 如果某个值只出现了一次，那么该下标的结果为 $-1$。
- 如果某个值出现了多次，遍历该值的所有下标，计算每个下标与其左右邻居的环形距离，取最小值。

**处理查询**：遍历 $queries$ 数组，直接从预计算好的结果中取值。

代码如下，已附加注释：

```Python
# python
class Solution:
    def solveQueries(self, nums: List[int], queries: List[int]) -> List[int]:
        n = len(nums)
        # 记录每个数字出现的下标
        pos_map = defaultdict(list)
        for i, val in enumerate(nums):
            pos_map[val].append(i)
        
        # 预计算每个下标对应的最小距离
        ans = [-1] * n
        
        for val, indices in pos_map.items():
            m = len(indices)
            if m <= 1:
                # 该数字只出现一次
                continue
            
            # 遍历该数字出现的所有下标
            for k in range(m):
                cur_idx = indices[k]
                
                # 环形下的左邻居和右邻居
                left_neighbor_idx = indices[(k - 1) % m]
                right_neighbor_idx = indices[(k + 1) % m]
                
                # 到左邻居的距离
                dist_left_abs = abs(cur_idx - left_neighbor_idx)
                dist_left = min(dist_left_abs, n - dist_left_abs)
                
                # 到右邻居的距离
                dist_right_abs = abs(cur_idx - right_neighbor_idx)
                dist_right = min(dist_right_abs, n - dist_right_abs)
                
                # 最小
                ans[cur_idx] = min(dist_left, dist_right)
        
        return [ans[q] for q in queries]
```

```Java
// java
class Solution {
    public List<Integer> solveQueries(int[] nums, int[] queries) {
        int n = nums.length;
        // 记录每个数字出现的下标
        Map<Integer, List<Integer>> posMap = new HashMap<>();
        for (int i = 0; i < n; i++) {
            posMap.computeIfAbsent(nums[i], k -> new ArrayList<>()).add(i);
        }

        // 预计算每个下标对应的最小距离
        int[] ans = new int[n];
        Arrays.fill(ans, -1);

        for (Map.Entry<Integer, List<Integer>> entry : posMap.entrySet()) {
            List<Integer> indices = entry.getValue();
            int m = indices.size();
            if (m <= 1) {
                // 该数字只出现一次
                continue;
            }

            // 遍历该数字出现的所有下标
            for (int k = 0; k < m; k++) {
                int curIdx = indices.get(k);

                // 环形下的左邻居和右邻居
                int leftNeighborIdx = indices.get((k - 1 + m) % m);
                int rightNeighborIdx = indices.get((k + 1) % m);

                // 到左邻居的距离
                int distLeftAbs = Math.abs(curIdx - leftNeighborIdx);
                int distLeft = Math.min(distLeftAbs, n - distLeftAbs);

                // 到右邻居的距离
                int distRightAbs = Math.abs(curIdx - rightNeighborIdx);
                int distRight = Math.min(distRightAbs, n - distRightAbs);

                // 最小
                ans[curIdx] = Math.min(distLeft, distRight);
            }
        }

        List<Integer> res = new ArrayList<>();
        for (int q : queries) {
            res.add(ans[q]);
        }
        return res;
    }
}
```

- 时间复杂度： $O(n+q)$，其中 $n,q$ 是数组 $words,queries$ 的长度，虽然是双重 $for$ 循环，但是每个下标只会被访问常数次
- 空间复杂度： $O(n)$，表示哈希表 $posMap$ 以及数组 $ans$ 的大小
