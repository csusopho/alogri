[Problem: 1848. 到目标元素的最小距离](https://leetcode.cn/problems/minimum-distance-to-the-target-element/description/)

### 方法：模拟

注意，题目保证 $target$ 一定存在，所以无需考虑边界检查等问题！

**最简单** 的想法：一次遍历，

- 检查当前数值 $num$ 是否等于 $target$
- 如果相等，计算当前下标与 $start$ 的绝对距离： $distance = abs(i - start)$
- 将计算出的 $distance$ 与当前记录的最小距离进行比较，保留较小的那一个
- 如果不相等，就跳过当前 $num$

遍历结束后，得到的最小值即为答案。

代码如下，已附加注释：

```Python
# python
class Solution:
    def getMinDistance(self, nums: List[int], target: int, start: int) -> int:
        ans = float('inf')
        
        for i, num in enumerate(nums):
            # 找到
            if num == target:
                # 计算
                cur_dist = abs(i - start)
                # 更小
                if cur_dist < ans:
                    ans = cur_dist
        
        return ans
```

```Java
// java
class Solution {
    public int getMinDistance(int[] nums, int target, int start) {
        int ans = Integer.MAX_VALUE;
        
        for (int i = 0; i < nums.length; i++) {
            // 找到
            if (nums[i] == target) {
                // 计算
                int curDist = Math.abs(i - start);
                // 更小
                if (curDist < ans) {
                    ans = curDist;
                }
            }
        }
        
        return ans;
    }
}
```

- 时间复杂度： $O(n)$，其中 $n$ 是数组 $nums$ 的长度
- 空间复杂度： $O(1)$，仅需要常数级别的额外空间

上面是从左到右的写法，本题也可以使用双指针，从 $start$ 的位置向左右两边搜索。不过，都是花费 $0ms$ 超过 $100\%$，这里不再展示。
