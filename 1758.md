[Problem: 1758. 生成交替二进制字符串的最少操作数](https://leetcode.cn/problems/minimum-changes-to-make-alternating-binary-string/description/)

### 方法：模拟 & 优化

仔细思考一下，对于任意长度为 $n$ 的交替字符串，其实 **只有两种形态**：

1. 形态一（以 $0$ 开头）：比如 $01010101...$，偶数索引位置是 $0$，奇数索引位置是 $1$
2. 形态二（以 $1$ 开头）：比如 $10101010...$，偶数索引位置是 $1$，奇数索引位置是 $0$

因此，问题转换成了：计算当前字符串 $s$ 变成 **“形态一”** 需要变几个字符，变成 **“形态二”** 需要变几个字符，然后取两者中的最小值。

观察形态一和形态二，我们会发现：在任何一个固定的索引 $i$，形态一和形态二的字符必定是 **相反的**。也即

- 如果 $s[i]$ 和形态一相同，那它必定和形态二不同。
- 如果 $s[i]$ 和形态一不同，那它必定和形态二相同。

**数学推论：** 如果字符串总长度为 $n$，把 $s$ 变成形态一需要的操作数为 $cnt1$，那么把 $s$ 变成形态二需要的操作数 $cnt2$ 直接等于 $n - cnt1$。

**结论：** 只需要遍历一次，统计出字符串 $s$ 变成 **其中任意一种形态** 的步数 $cnt1$，然后直接返回 $min(cnt1, n - cnt1)$ 即可。

代码如下，已附加注释：

```Python
# python
class Solution:
    def minOperations(self, s: str) -> int:
        cnt1 = 0  # 变成 010 需要的操作数
        n = len(s)
        
        for i, char in enumerate(s):
            # 偶数索引应为 0，奇数索引应为 1
            expected = str(i % 2)
            # 不同，说明需要操作一次
            if char != expected:
                cnt1 += 1
        
        # 形态二的操作数是 n - cnt1
        return min(cnt1, n - cnt1)
```

```Java
// java
class Solution {
    public int minOperations(String s) {
        int cnt1 = 0; // 变成 010 需要的操作数
        int n = s.length();
        
        for (int i = 0; i < n; i++) {
            char c = s.charAt(i);
            // 偶数索引应为 0，奇数索引应为 1
            char expected = (char) (i % 2 + '0');
            // 不同，说明需要操作一次
            if (c != expected) {
                cnt1++;
            }
        }
        
        // 形态二的操作数是 n - cnt1
        return Math.min(cnt1, n - cnt1);
    }
}
```

- 时间复杂度： $O(n)$，其中 $n$ 是字符串 $s$ 的长度
- 空间复杂度： $O(1)$，仅使用常数个额外变量

### 优化

根据索引的奇偶性，判断当前字符，有些繁琐。

其中只需要分开遍历两次，

- 遍历所有 **偶数** 索引，这个位置应该是 $1$，统计错误变为 $0$ 的数量 $cnt0$
- 遍历所有 **奇数** 索引，这个位置应该是 $0$，统计错误变为 $1$ 的数量 $cnt1$

变为 “形态二” $101$ 需要的总操作次数，就是修改上面每一个位置的次数，也即 $cnt0+cnt1$。

这种写法速度更快，代码如下，已附加注释：

```Python
class Solution:
    def minOperations(self, s: str) -> int:
        # 统计偶数位上变成了 0 的错误数量
        even_zeros = s[::2].count("0")
        # 统计奇数位上变成了 1 的错误数量
        odd_ones = s[1::2].count("1")
        
        # 变成 1010 的总操作数
        cost_10 = even_zeros + odd_ones
        total_len = len(s)
        
        return min(cost_10, total_len - cost_10)
```

```Java
// java
class Solution {
    public int minOperations(String s) {
        // 统计偶数位上变成了 0 的错误数量
        int even_zeros = 0;
        for (int i = 0; i < s.length(); i += 2) {
            if (s.charAt(i) == '0') {
                even_zeros++;
            }
        }
        
        // 统计奇数位上变成了 1 的错误数量
        int odd_ones = 0;
        for (int i = 1; i < s.length(); i += 2) {
            if (s.charAt(i) == '1') {
                odd_ones++;
            }
        }
        
        // 变成 1010 的总操作数
        int cost_10 = even_zeros + odd_ones;
        int total_len = s.length();
        
        return Math.min(cost_10, total_len - cost_10);
    }
}
```

- 时间复杂度： $O(n)$
- 空间复杂度： $O(1)$
