[Problem: 3464. 正方形上的点之间的最大距离](https://leetcode.cn/problems/maximize-the-distance-between-points-on-a-square/description/)

### 方法：二分

#### 拆解题意

什么叫做最小距离？

考虑一维情况（首尾相通），假设总共 $4$ 个点，坐标分别为 $1,3,5,8$。

- 不考虑首尾坐标，两两之间的距离为 $2,2,3$。最小距离为 $2$；
- 假设首尾相接，且为环形结构，首尾距离为 $1$，那么最小距离为 $1$。

什么叫做最大化最小？

- 第一次选择出 $k$ 个元素，它们之间的最小距离为 $2$；
- 第二次选择出 $k$ 个元素，它们之间的最小距离为 $9$；
- $\dots$

那么，上述所有最小距离中，最大值是 $9$。

---

#### 思路分析

根据上述推导，最好首先将正方形边界映射为 **一维坐标**：

正方形边界上的任何一点，都可以通过从左下角 $(0,0)$ 开始逆时针行走的距离 $p$ 唯一表示。周长为 $4 \times side$，给定任意 $p \in [0, 4 \times side)$：

- 当 $p \in [0, side]$，点在底边，坐标为 $(p, 0)$。
- 当 $p \in [side, 2 \times side]$，点在右边，坐标为 $(side, p - side)$。
- 当 $p \in [2 \times side, 3 \times side]$，点在顶边，坐标为 $(3 \times side - p, side)$。
- 当 $p \in [3 \times side, 4 \times side]$，点在左边，坐标为 $(0, 4 \times side - p)$。

这样，我们将二维点的选取问题转化为了在周长为 $4 \times side$ 的环上选取 $k$ 个点。

此时，再看到 “最大化最小值”，会想到使用 **二分查找**。

也即，不用去计算怎样选点才是最佳的，而是去 **猜** 一个距离 $m$。转化思考：“如果要求任意两点间距离都 $\ge m$，我能选出 $k$ 个点吗？”

- 如果能，说明 $m$ 还可以更大（扩大下界）
- 如果不能，说明 $m$ 猜大了（缩小上界）

这就是二分的思路由来。

---

#### 难点剖析

对于任意的点 $A$，如何找出与它距离距离 $\ge m$ 的下一个点？

注意，如果 $m \le side$，那么边界上两点的曼哈顿距离 **等于** 它们在边界上的最短路径距离。当 $k \ge 4$ 时，鸽巢原理保证了最小距离一定 $\le side$，此时曼哈顿距离完全退化为环上距离。

但对于 $k = 2$ 或 $k = 3$ 时，最小距离可能 $> side$。这种情况下，曼哈顿距离与环上距离不完全等价，甚至随着点在环上移动，距离并不是单调变化的。

**核心观察**：对于固定的点 $A$，与它曼哈顿距离 $\ge m$ 的点域，在边界环上一定构成 **一段连续的有效环形区间**。

有点难理解？想象一共有 $4$ 个点，它们分布在四条边上。对于下边这个点来说，与它距离 $\ge side$ 的点一定是在正方形的上半区域。如果点位于 $(0,0)$，且其他点足够密集足够多的话，它们能连线成上边和右边。

> $ps$：这里不画图了，麻烦各位脑补~

上面这段话是什么作用？

在一条直线上，距离越远，直线距离就越长。但在正方形上，**曼哈顿距离并不等于你沿着边界走的距离**。没有单调性！无法二分！

因此，要在环上找到下一个合法的点，不能简单地在 $1D$ 坐标上加上 $m$，而是必须通过 **解绝对值不等式**，精确计算出正方形四条边上的哪些区间是合法的。

---

#### 公式推导

给定点 $A$ 和要求的距离 $m$，求出在某一条边上，哪些区间内的点 $B$ 满足与 $A$ 的曼哈顿距离 $\ge m$？

假设点 $A$ 的二维坐标是 $(x_A, y_A)$。目前测试正方形的 **底边**。底边上的点 $B$ 的特征是：横坐标是 $x \in [0, side]$，纵坐标是 $y = 0$。

$A$ 和 $B$ 的曼哈顿距离公式为：

$$
Dist = |x - x_A| + |0 - y_A| = |x - x_A| + y_A
$$

我们要求这个距离 $\ge m$，即：

$$
|x - x_A| + y_A \ge m \implies |x - x_A| \ge m - y_A
$$

为了通用化，令 $C = x_A$， $D = m - y_A$。不等式变成了标准的绝对值不等式：

$$
|x - C| \ge D
$$

**解这个不等式分两种情况：**

- 如果 $D \le 0$：

由于绝对值总是 $\ge 0$ 的，所以对于底边上所有的 $x \in [0, side]$ 都成立！对应的代码就是：`intervals.append([offset, offset + side])`。

- 如果 $D > 0$：

拆解绝对值，得到 $x - C \ge D$ 或 $x - C \le -D$。因为点必须在这条边上，所以 $x$ 还要受限于区间 $[0, side]$。

**左半部分合法区间**：从 $0$ 开始，到 $C - D$ 结束，且不能超过 $side$。前提是 $C - D \ge 0$。对应代码：`intervals.append([offset, offset + min(side, C - D)])`

**右半部分合法区间**：从 $C + D$ 开始，到 $side$ 结束。且起点不能小于 $0$。前提是 $C + D \le side$。对应代码：`intervals.append([offset + max(0, C + D), offset + side])`

对于其他三条边（右、顶、左），推导逻辑完全一样，只是公式里的 $C$ 和 $D$ 变了，以及区间起点 $offset$ 加上了对应的偏置 $1/2/3\times side$。

---

#### 二分逻辑

对于所有的点 $i$，使用上面这个函数，算出所有合法的区间，然后将这些区间合并。

接着，使用二分查找，在这些合法区间内寻找 **第一个** 存在的真实点的下标。记录为 $nxt[i]$。这相当于建立了一张有向图：`i -> nxt[i]` 表示点 $i$ 往后跳的第一步合法落脚点。

接着，使用 **贪心** 维护单调性。`nxt[i] = min(nxt[i], nxt[i + 1])`

为什么要这一步？假设 `nxt[2] = 5`，而 `nxt[3] = 4`。说明从点 $2$ 出发，因为曼哈顿距离的怪异性，下一个合法点是点 $5$；而从点 $3$ 出发，下一个合法点反而只需要到点 $4$。

那么从点 $2$ 出发，我们 **完全可以直接跳到点 4**，相当于把起点推迟到点 $3$ 再跳，这样找全 $k$ 个点会更容易。这一步让 $nxt$ 数组单调不降，保证了我们使用的贪心策略必定最优。

接着，试探起点。如果能选出 $k$ 个合法的点，那么总有一个合法的序列，它的 **起点** 一定包含在区间 $[0, nxt[0]]$ 之间！

反证法：如果起点超过了 $nxt[0]$，我们完全可以把那个起点替换为点 $0$，后续的路径依旧合法，所以只需检验到 $nxt[0]$ 即可。

至此，我们可以验证假设的距离 $m$ 是否可行。

在主流程中，使用闭区间二分，寻找合适的 $m$。

代码如下，已附加注释：

```Python
# python
class Solution:
    def maxDistance(self, side: int, points: List[List[int]], k: int) -> int:
        # 转化为 1D 坐标
        coords = []
        for x, y in points:
            if y == 0:
                coords.append(x)
            elif x == side:
                coords.append(side + y)
            elif y == side:
                coords.append(3 * side - x)
            elif x == 0:
                coords.append(4 * side - y)
         
        coords.sort()
        n = len(coords)
        # 将环展开两倍
        extended_coords = coords + [p + 4 * side for p in coords]
        
        # 将一维坐标转回二维坐标
        def get_xy(p):
            p %= (4 * side)
            if p <= side:
                return p, 0
            if p <= 2 * side:
                return side, p - side
            if p <= 3 * side:
                return 3 * side - p, side
            return 0, 4 * side - p
        
        # 计算两个一维坐标的曼哈顿距离
        def manhattan_dist(p1, p2):
            x1, y1 = get_xy(p1)
            x2, y2 = get_xy(p2)
            return abs(x1 - x2) + abs(y1 - y2)
        
        # 二分，验证是否存在最小距离 >= m 的 k 个点
        def check(m):
            nxt = [2 * n] * (2 * n + 1)
            
            # 为每个点寻找距离 >= m 的下一个合法点
            for i in range(2 * n):
                xA, yA = get_xy(extended_coords[i])
                intervals = []
                
                # 收集在各条边上满足距离 >= m 的一维有效区间
                def add_valid_intervals(C, D, offset):
                    if D <= 0:
                        intervals.append([offset, offset + side])
                    else:
                        if C - D >= 0:
                            intervals.append([offset, offset + min(side, C - D)])
                        if C + D <= side:
                            intervals.append([offset + max(0, C + D), offset + side])
                            
                add_valid_intervals(xA, m - yA, 0)                 # 底边
                add_valid_intervals(yA, m - side + xA, side)       # 右边
                add_valid_intervals(side - xA, m - side + yA, 2 * side) # 上边
                add_valid_intervals(side - yA, m - xA, 3 * side)   # 左边
                
                # 合并重叠的合法区间
                intervals.sort()
                merged = []
                for u, v in intervals:
                    if not merged or u > merged[-1][1]:
                        merged.append([u, v])
                    else:
                        merged[-1][1] = max(merged[-1][1], v)
                
                # 映射到扩展后的坐标系中
                target = extended_coords[i] + 1
                best_j = 2 * n
                for k_shift in [0, 4 * side, 8 * side]:
                    for u, v in merged:
                        shifted_u, shifted_v = u + k_shift, v + k_shift
                        if shifted_v >= target:
                            # 二分查找落在合法区间内的第一个给定点
                            search_val = max(shifted_u, target)
                            idx = bisect_left(extended_coords, search_val)
                            if idx < 2 * n and extended_coords[idx] <= shifted_v:
                                best_j = min(best_j, idx)
                    # 只要找到了下一个点，后续的偏移范围无需再找
                    if best_j < 2 * n:
                        break
                nxt[i] = best_j
            
            # 倒序遍历，维护 nxt 数组的单调性
            for i in range(2 * n - 1, -1, -1):
                nxt[i] = min(nxt[i], nxt[i + 1])
            
            # 试探区间 [0, nxt[0]] 内的起点
            limit = min(n, nxt[0] + 1)
            for i in range(limit):
                curr = i
                for _ in range(k - 1):
                    curr = nxt[curr]
                    if curr >= 2 * n:
                        break
                # 跳跃 k-1 次后，验证最终落点与起点
                if curr < i + n and manhattan_dist(extended_coords[i], extended_coords[curr]) >= m:
                    return True
            return False
        
        # 二分，寻找可能的最大曼哈顿距离
        low, high = 0, 2 * side
        ans = 0
        while low <= high:
            mid = (low + high) // 2
            if check(mid):
                ans = mid
                low = mid + 1  # 尝试更大的距离
            else:
                high = mid - 1
        
        return ans
```

```Java
// java
class Solution {
    public int maxDistance(int side, int[][] points, int k) {
        // 转化为 1D 坐标
        int n = points.length;
        long[] coords = new long[n];
        for (int i = 0; i < n; i++) {
            long x = points[i][0];
            long y = points[i][1];
            if (y == 0) {
                coords[i] = x;
            } else if (x == side) {
                coords[i] = side + y;
            } else if (y == side) {
                coords[i] = 3L * side - x;
            } else if (x == 0) {
                coords[i] = 4L * side - y;
            }
        }
        
        Arrays.sort(coords);
        
        // 将环展开两倍
        long[] extended_coords = new long[2 * n];
        for (int i = 0; i < n; i++) {
            extended_coords[i] = coords[i];
            extended_coords[i + n] = coords[i] + 4L * side;
        }
        
        // 二分，寻找可能的最大曼哈顿距离
        long low = 0, high = 2L * side;
        long ans = 0;
        while (low <= high) {
            long mid = low + (high - low) / 2;
            if (check(mid, extended_coords, n, side, k)) {
                ans = mid;
                low = mid + 1;  // 尝试更大的距离
            } else {
                high = mid - 1;
            }
        }
        
        return (int) ans;
    }

    // 二分，验证是否存在最小距离 >= m 的 k 个点
    private boolean check(long m, long[] extended_coords, int n, int side, int k) {
        int[] nxt = new int[2 * n + 1];
        Arrays.fill(nxt, 2 * n);
        
        // 为每个点寻找距离 >= m 的下一个合法点
        for (int i = 0; i < 2 * n; i++) {
            long[] xy = get_xy(extended_coords[i], side);
            long xA = xy[0], yA = xy[1];
            List<long[]> intervals = new ArrayList<>();
            
            add_valid_intervals(xA, m - yA, 0, side, intervals);                 // 底边
            add_valid_intervals(yA, m - side + xA, side, side, intervals);       // 右边
            add_valid_intervals(side - xA, m - side + yA, 2L * side, side, intervals); // 上边
            add_valid_intervals(side - yA, m - xA, 3L * side, side, intervals);   // 左边
            
            // 合并重叠的合法区间
            intervals.sort((a, b) -> Long.compare(a[0], b[0]));
            List<long[]> merged = new ArrayList<>();
            for (long[] interval : intervals) {
                long u = interval[0];
                long v = interval[1];
                if (merged.isEmpty() || u > merged.get(merged.size() - 1)[1]) {
                    merged.add(new long[]{u, v});
                } else {
                    merged.get(merged.size() - 1)[1] = Math.max(merged.get(merged.size() - 1)[1], v);
                }
            }
            
            // 映射到扩展后的坐标系中
            long target = extended_coords[i] + 1;
            int best_j = 2 * n;
            long[] shifts = {0, 4L * side, 8L * side};
            for (long k_shift : shifts) {
                for (long[] interval : merged) {
                    long shifted_u = interval[0] + k_shift;
                    long shifted_v = interval[1] + k_shift;
                    if (shifted_v >= target) {
                        // 二分查找落在合法区间内的第一个给定点
                        long search_val = Math.max(shifted_u, target);
                        int idx = bisect_left(extended_coords, search_val);
                        if (idx < 2 * n && extended_coords[idx] <= shifted_v) {
                            best_j = Math.min(best_j, idx);
                        }
                    }
                }
                // 只要找到了下一个点，后续的偏移范围无需再找
                if (best_j < 2 * n) {
                    break;
                }
            }
            nxt[i] = best_j;
        }
        
        // 倒序遍历，维护 nxt 数组的单调性
        for (int i = 2 * n - 1; i >= 0; i--) {
            nxt[i] = Math.min(nxt[i], nxt[i + 1]);
        }
        
        // 试探区间 [0, nxt[0]] 内的起点
        int limit = Math.min(n, nxt[0] + 1);
        for (int i = 0; i < limit; i++) {
            int curr = i;
            for (int j = 0; j < k - 1; j++) {
                curr = nxt[curr];
                if (curr >= 2 * n) {
                    break;
                }
            }
            // 跳跃 k-1 次后，验证最终落点与起点
            if (curr < i + n && manhattan_dist(extended_coords[i], extended_coords[curr], side) >= m) {
                return true;
            }
        }
        return false;
    }

    // 将一维坐标转回二维坐标
    private long[] get_xy(long p, int side) {
        p %= (4L * side);
        if (p <= side) {
            return new long[]{p, 0};
        }
        if (p <= 2L * side) {
            return new long[]{side, p - side};
        }
        if (p <= 3L * side) {
            return new long[]{3L * side - p, side};
        }
        return new long[]{0, 4L * side - p};
    }

    // 计算两个一维坐标的曼哈顿距离
    private long manhattan_dist(long p1, long p2, int side) {
        long[] p1_xy = get_xy(p1, side);
        long[] p2_xy = get_xy(p2, side);
        return Math.abs(p1_xy[0] - p2_xy[0]) + Math.abs(p1_xy[1] - p2_xy[1]);
    }

    // 收集在各条边上满足距离 >= m 的一维有效区间
    private void add_valid_intervals(long C, long D, long offset, int side, List<long[]> intervals) {
        if (D <= 0) {
            intervals.add(new long[]{offset, offset + side});
        } else {
            if (C - D >= 0) {
                intervals.add(new long[]{offset, offset + Math.min((long) side, C - D)});
            }
            if (C + D <= side) {
                intervals.add(new long[]{offset + Math.max(0L, C + D), offset + side});
            }
        }
    }

    private int bisect_left(long[] arr, long target) {
        int left = 0, right = arr.length;
        while (left < right) {
            int mid = left + (right - left) / 2;
            if (arr[mid] < target) {
                left = mid + 1;
            } else {
                right = mid;
            }
        }
        return left;
    }
}
```

- 时间复杂度： $O(n\times logn\times logD)$，其中 $n$ 是数组 $points$ 的长度，快速排序，二分最多循环 $D=\log_2(2 \times 10^9) \approx 31$ 次。在 $check$ 函数内，每次二分时间为 $O(logn)$，维护单调性时间为 $O(n)$。跳跃验证：起点数量 $\times$ 跳跃次数 $k$。由于起点的范围被限制在 $[0, nxt[0]]$，这部分点的数量不会超过 $\lceil \frac{n}{k} \rceil$，总次数严格受限于 $O(n)$。
- 空间复杂度： $O(n)$，表示 $coords,nxt$ 等数组的大小
