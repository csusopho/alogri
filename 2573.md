[Problem: 2573. 找出对应 LCP 矩阵的字符串](https://leetcode.cn/problems/find-the-string-with-lcp/description/)

### 方法：并查集

什么叫做最长公共前缀 $LCP$？比如串 $\textbf{abd} \text{egfdg}$ 与串 $\textbf{abd} \text{fdfef}$ 的公共前缀有 $a,ab,abd$，最长的是 $abd$。

题意有一点绕，本质只有一个串 $word$，它的子串 $[i,n-1]$ 与子串 $[j,n-1]$ 计算出一个最长公共前缀，然后放在 $lcp[i][j]$ 位置。也就是说，我们需要根据 $LCP$ 逆推出 $word$。

**LCP 矩阵有什么性质？** 对于字符串 $word$：

1. 对称性：两个子串互相比较，那么 $lcp[i][j]$ 必须等于 $lcp[j][i]$。
2. 对角线：由于 $lcp[i][i]$ 是后缀 $word[i \dots n-1]$ 与自身的 $LCP$，其长度固定为 $n-i$。
3. 传递性/等价关系：如果 $lcp[i][j] > 0$，说明有公共部分，则 **第一个字符** $word[i]$ 必须等于 $word[j]$。这意味着，可以将所有满足 $lcp[i][j] > 0$ 的下标索引 $i$ 和 $j$ 看作在一个 **连通分量** 里，它们必须共享同一个字母。

构造出 $word$ 后，如何正向计算出 $LCP$？有一个 **递推 $DP$ 关系**：

- 如果 $word[i] = word[j]$，则 $lcp[i][j] = 1 + lcp[i+1][j+1]$
- 如果 $word[i] \ne word[j]$，则 $lcp[i][j] = 0$

什么？ $DP$ 原理不好懂？可以类比为 “从后往前比较两个序列”。

假设我们要计算字符串 $word$ 中两个后缀 $S_i$ 和 $S_j$ 的最长公共前缀长度。

- 如果 **首字母** 不同，即 $word[i] \neq word[j]$，它们绝对不可能有共同的前缀。

$$
\text{LCP}(i, j) = 0
$$

- 如果 **首字母** 相同，即 $word[i] = word[j]$，它们至少有 $1$ 个公共字符。剩下的公共部分，就取决于它们 **各自往后移一位** 后的最长公共前缀。

$$
\text{LCP}(i, j) = 1 + \text{LCP}(i+1, j+1)
$$

为什么需要从后往前（倒序）计算？

观察上面的公式：计算 $LCP[i][j]$ 需要用到 $LCP[i+1][j+1]$ 的值。也就是说，当前状态依赖于 “右下方” 的状态，所以从后往前填充。

---

根据上面的分析，我们大致得出本题思路：

- 首先检查 $lcp[i][j]$ 是否符合矩阵的上面列出的几个基本性质，提前剪枝
- 位置 $i$ 和 $j$ 拥有相同的字符时，使用并查集将这些下标聚合到一起
- 贪心构造
- 使用动态规划判断构造出的 $word$ 是否符合要求

如何 **贪心** 构造？为了获得字典序最小的字符串：

- 如果当前 $i$ 尚未分配字符，需要为 $i$ 所在的整个并查集集合分配一个字符。
- 依次尝试 $'a'$ 到 $'z'$。
- 如果选择了字符 $c$，必须满足：对于所有已经分配了字符的 $j < i$，如果 $lcp[i][j] = 0$，则 $word[j]$ 不能是 $c$。因为如果 $word[i]$ 和 $word[j]$ 都是 $c$，那么它们的 $LCP$ 至少为 1。而 $lcp[i][j] = 0$，则要求它们必须是不同字符。
- 如果 $26$ 个字母都不能用，说明无解。

这就是下面代码中 $forbidden$ 的逻辑，先看看之前已经填好的字符里，哪些是冲突的，放入 $forbidden$ 集合中，然后从 $'a'$ 到 $'z'$ 挑一个没被禁用的。

**切记**：贪心构造只利用了 $lcp[i][j] > 0$ 和 $lcp[i][j] = 0$ 的逻辑，并不能保证 $lcp[i][j]$ 的具体数值（如 $2$ 或 $3$）完全匹配！

我们需要验证 $word$，利用递推 $DP$ 公式重新计算其 $LCP$ 矩阵，并与原始输入 $lcp$ 比对。如果这个公式推导出的每个值都和输入矩阵对得上，才说明构造正确。

```Python
# python
class Solution:
    def findTheString(self, lcp: List[List[int]]) -> str:
        n = len(lcp)
        # 并查集初始化
        parent = list(range(n))
        
        def find(i):
            if parent[i] == i:
                return i
            # 路径压缩
            parent[i] = find(parent[i])
            return parent[i]
        
        def union(i, j):
            root_i, root_j = find(i), find(j)
            if root_i != root_j:
                parent[root_i] = root_j
        
        for i in range(n):
            if lcp[i][i] != n - i:  # 对角线检查
                return ""
            for j in range(n):
                if lcp[i][j] != lcp[j][i]:  # 对称性检查
                    return ""
                if lcp[i][j] > n - max(i, j):  # 长度越界检查
                    return ""
                if lcp[i][j] > 0:  # 并查集分组
                    union(i, j)
        
        # 贪心构造
        res = [None] * n
        root_to_char = {}
        
        for i in range(n):
            root = find(i)
            if root in root_to_char:  # 在同一组
                res[i] = root_to_char[root]
            else:
                # 找出所有与当前下标 i 冲突的已分配字符
                forbidden = set()
                for j in range(i):
                    if lcp[i][j] == 0:
                        forbidden.add(res[j])
                
                # 分配最小的可用字母
                assigned = False
                for char_code in range(ord('a'), ord('z') + 1):
                    char = chr(char_code)
                    if char not in forbidden:
                        root_to_char[root] = char
                        res[i] = char
                        assigned = True
                        break
                
                if not assigned:  # 超过26个字母
                    return ""
        
        word = "".join(res)
        
        # 动态规划验证
        actual_lcp = [[0] * (n + 1) for _ in range(n + 1)]
        for i in range(n - 1, -1, -1):
            for j in range(n - 1, -1, -1):
                if word[i] == word[j]:
                    actual_lcp[i][j] = actual_lcp[i+1][j+1] + 1
                else:
                    actual_lcp[i][j] = 0
                
                # 与给定值不符
                if actual_lcp[i][j] != lcp[i][j]:
                    return ""
        
        return word
```

```Java
// java
class Solution {
    public String findTheString(int[][] lcp) {
        int n = lcp.length;
        // 并查集初始化
        int[] parent = new int[n];
        for (int i = 0; i < n; i++) {
            parent[i] = i;
        }
        
        for (int i = 0; i < n; i++) {
            if (lcp[i][i] != n - i) { // 对角线检查
                return "";
            }
            for (int j = 0; j < n; j++) {
                if (lcp[i][j] != lcp[j][i]) { // 对称性检查
                    return "";
                }
                if (lcp[i][j] > n - Math.max(i, j)) { // 长度越界检查
                    return "";
                }
                if (lcp[i][j] > 0) { // 并查集分组
                    union(i, j, parent);
                }
            }
        }

        // 贪心构造
        char[] res = new char[n];
        char[] rootToChar = new char[n]; // 根节点到字符的映射

        for (int i = 0; i < n; i++) {
            int root = find(i, parent);
            if (rootToChar[root] != 0) { // 在同一组
                res[i] = rootToChar[root];
            } else {
                // 找出所有与当前下标 i 冲突的已分配字符
                boolean[] forbidden = new boolean[26];
                for (int j = 0; j < i; j++) {
                    if (lcp[i][j] == 0) {
                        forbidden[res[j] - 'a'] = true;
                    }
                }

                // 分配最小的可用字母
                boolean assigned = false;
                for (int c = 0; c < 26; c++) {
                    if (!forbidden[c]) {
                        char charToAssign = (char) ('a' + c);
                        rootToChar[root] = charToAssign;
                        res[i] = charToAssign;
                        assigned = true;
                        break;
                    }
                }

                if (!assigned) { // 超过26个字母
                    return "";
                }
            }
        }

        String word = new String(res);

        // 动态规划验证
        int[][] actualLcp = new int[n + 1][n + 1];
        for (int i = n - 1; i >= 0; i--) {
            for (int j = n - 1; j >= 0; j--) {
                if (res[i] == res[j]) {
                    actualLcp[i][j] = actualLcp[i + 1][j + 1] + 1;
                } else {
                    actualLcp[i][j] = 0;
                }

                // 与给定值不符
                if (actualLcp[i][j] != lcp[i][j]) {
                    return "";
                }
            }
        }

        return word;
    }

    private int find(int i, int[] parent) {
        if (parent[i] == i) {
            return i;
        }
        // 路径压缩
        return parent[i] = find(parent[i], parent);
    }

    private void union(int i, int j, int[] parent) {
        int rootI = find(i, parent);
        int rootJ = find(j, parent);
        if (rootI != rootJ) {
            parent[rootI] = rootJ;
        }
    }
}
```

单次「合并」与「查询」操作的时间复杂度为 $Ackermann$ 函数 的反函数，记为 $\alpha$，也即一次合并或查询为 $O(\alpha(n))$，在实际应用中可以认为是常数时间。

- 时间复杂度： $O(n^2)$，其中 $n$ 是串 $word$ 的长度，并查集分组时间为 $O(n^2\cdot \alpha(n))$，字符串的构造与验证时间为 $O(n^2)$
- 空间复杂度： $O(n^2)$，表示动规表 $actual$ 的大小
