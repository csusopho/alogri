[Problem: 1888. 使二进制字符串字符交替的最少反转次数](https://leetcode.cn/problems/minimum-number-of-flips-to-make-the-binary-string-alternating/description/)

### 方法：滑窗 & 优化

操作 $1$ 是将字符串的 **第一个** 字符移到末尾，这相当于在 **循环** 使用原字符串。

假设字符串是 $s = 111000$，将它拼接一次变成 $double = s + s = 111000111000$。在这个长字符串中，所有长度为 $n$ 的连续子串，正好对应了所有执行若干次操作 $1$ 后的结果：

- 移动 $0$ 次：**111000** 111000
- 移动 $1$ 次：1 **110001** 11000
- 移动 $2$ 次：11 **100011** 1000
- ...以此类推

操作 $2$ 是翻转字符。如何让子串变成交替字符串？需要的操作次数就是 “当前窗口字符串与目标字符串 **不同的字符个数**”。

举个例子，串 $1001$ 想变为 $1010$，就是修改后两个不同字符，操作次数为 $2$。

如果每次移动窗口后，都重新遍历一次窗口，计算不同字符的数量，时间复杂度会是 $O(n^2)$。已知新窗口 $s[i+1,j+1]$ 与原窗口 $s[i,j]$ 只是相差首尾字符，因此，使用 **常规优化**：

- 使用一个全局变量 $diff$ 维护当前窗口的总差异（操作）数。
- 窗口向右滑动一格，意味着 **离开一个旧字符，进入一个新字符**。
- 如果离开的旧字符原本与目标字符串不同，说明差异减少了，差异数 $-1$。
- 如果进入的新字符与目标字符串不同，说明差异增加了，差异数 $+1$。

> 小经验：循环串的问题，一般都是重复拼接+滑动窗口解决！

1）预处理：把原字符串 $s$ 拼接成 $double = s + s$。同时，构造两个长度为 $2n$ 的标准交替字符串，用来作为对比的 “标尺”：

- $target0 = 010101010101...$
- $target1 = 101010101010...$

2）初始时：取 $double$ 的前 $n$ 个字符作为 **第一个窗口**。分别对比它与 $target0$ 和 $target1$ 前 $n$ 个字符的差异（不同字符的个数）。记录下这时的差异数 $diff0$ 和 $diff1$。

3）开始滑动：窗口大小固定为 $n$，每次整体向右滑动 $1$ 步，直到滑到 $double$ 的末尾：

- **旧字符出窗口**：左侧 $double[i-n]$ 会离开窗口，检查该字符，如果它和对应的标尺不一样，现在它离开窗口，差异数要减 $1$。
- **新字符进窗口**：右侧 $double[i]$ 会进入窗口，检查该字符，如果它和对应的标尺不一样，现在属于窗口，差异数要加 $1$。
- 每次滑动后，更新全局的最小反转次数 $ans$。

代码如下，已附加注释：

```Python
# python
class Solution:
    def minFlips(self, s: str) -> int:
        n = len(s)

        # 拼接字符串
        s_double = s + s
        
        # 长度为 2n 的目标交替标尺
        target_0 = "01" * n
        target_1 = "10" * n
                
        # 初始化，计算第一个窗口的差异数
        diff_0 = 0
        diff_1 = 0
        for i in range(n):
            if s_double[i] != target_0[i]:
                diff_0 += 1
            if s_double[i] != target_1[i]:
                diff_1 += 1
        
        ans = min(diff_0, diff_1)
        # 开始滑动
        for i in range(n, 2 * n):
            # 移出
            left_idx = i - n
            if s_double[left_idx] != target_0[left_idx]:
                diff_0 -= 1
            if s_double[left_idx] != target_1[left_idx]:
                diff_1 -= 1
            
            # 移入
            right_idx = i
            if s_double[right_idx] != target_0[right_idx]:
                diff_0 += 1
            if s_double[right_idx] != target_1[right_idx]:
                diff_1 += 1
            
            # 更新
            ans = min(ans, diff_0, diff_1)
        
        return ans
```

```Java
// java
class Solution {
    public int minFlips(String s) {
        int n = s.length();

        // 拼接字符串
        String sDouble = s + s;
        
        // 长度为 2n 的目标交替标尺
        String target0 = "01".repeat(n);
        String target1 = "10".repeat(n);
                
        // 初始化，计算第一个窗口的差异数
        int diff0 = 0;
        int diff1 = 0;
        for (int i = 0; i < n; i++) {
            if (sDouble.charAt(i) != target0.charAt(i)) {
                diff0 += 1;
            }
            if (sDouble.charAt(i) != target1.charAt(i)) {
                diff1 += 1;
            }
        }
        
        int ans = Math.min(diff0, diff1);
        // 开始滑动
        for (int i = n; i < 2 * n; i++) {
            // 移出
            int leftIdx = i - n;
            if (sDouble.charAt(leftIdx) != target0.charAt(leftIdx)) {
                diff0 -= 1;
            }
            if (sDouble.charAt(leftIdx) != target1.charAt(leftIdx)) {
                diff1 -= 1;
            }
            
            // 移入
            int rightIdx = i;
            if (sDouble.charAt(rightIdx) != target0.charAt(rightIdx)) {
                diff0 += 1;
            }
            if (sDouble.charAt(rightIdx) != target1.charAt(rightIdx)) {
                diff1 += 1;
            }
            
            // 更新
            ans = Math.min(ans, Math.min(diff0, diff1));
        }
        
        return ans;
    }
}
```

- 时间复杂度： $O(n)$，其中 $n$ 是字符串 $s$ 的长度
- 空间复杂度： $O(n)$，表示拼接串 $double$ 以及标尺 $target$ 的大小

### 优化

使用 $s+s$ 真的去拼接没什么必要，毕竟是循环串，可以用 $\%n$ 的方式取出相应下标。同理，也无需使用 $target$ 去构造目标交替串，判断下标的奇偶性，就能知道对应目标字符。

因此，结合 `%` 写出下面的代码：

```Python
class Solution:
    def minFlips(self, s: str) -> int:
        n = len(s)
        diff_0 = 0
        diff_1 = 0
        
        # 第一个窗口
        for i in range(n):
            expected_0 = '0' if i % 2 == 0 else '1'
            expected_1 = '1' if i % 2 == 0 else '0'
            
            if s[i] != expected_0:
                diff_0 += 1
            if s[i] != expected_1:
                diff_1 += 1
        
        ans = min(diff_0, diff_1)
        # 开始滑动
        for i in range(n, 2 * n):
            # 移出
            out_idx = i - n
            # 预期字符
            expected_0_out = '0' if out_idx % 2 == 0 else '1'
            expected_1_out = '1' if out_idx % 2 == 0 else '0'
            
            # 不匹配
            if s[out_idx % n] != expected_0_out:
                diff_0 -= 1
            if s[out_idx % n] != expected_1_out:
                diff_1 -= 1
            
            # 移入i，预期字符
            expected_0_in = '0' if i % 2 == 0 else '1'
            expected_1_in = '1' if i % 2 == 0 else '0'
            
            # 不匹配
            if s[i % n] != expected_0_in:
                diff_0 += 1
            if s[i % n] != expected_1_in:
                diff_1 += 1
            
            # 更新
            ans = min(ans, diff_0, diff_1)
        
        return ans
```

但是，上面这种方法不怎么好，速度比第一种慢上很多。本质相当于用时间换空间，而且代码也变得更冗长。所以，我查看了通过速度最快的代码，下面做个经验分享。

因为二进制交替串只有 $0101...$ 和 $1010...$ 两种。它们是 **完全互补** 的。如果我们算出了当前字符串与第一种的区别有 $k$ 处，那么它与第二种的区别肯定是 $n - k$ 处！

还是用 $1001$ 举例，它和第一种串区别为前两个字符，和第二种串区别为后两个字符。所以，我们从头到尾只需要追踪 $target = 01$ 这 **一个模式**，大大节省了判断。

$n$ 为偶数时，直接返回！比方说 $1100$，移到末尾变成 $1001$，可以发现对于目标 $0101$ 和 $1010$ 来说，匹配的数量只是互相调换了一下，但 **它们的最小值始终没变**。

无论把第一个字符移到末尾多少次，字符串内部元素的 **相对奇偶位置关系** 本质上没有改变。此时的翻转只会让与 $01...$ 和 $10...$ 的差异数互换，不会产生更小的 $min$ 值。所以偶数情况下，根本不需要执行操作 $1$ 滑动，初始的答案就是最终答案。

下面是正常的滑动窗口。因为 $n$ 是奇数，所以 $i$ 和 $i+n$ 的 **奇偶性必然相反**。原来的字符该和 $0$ 对比，现在就该和 $1$ 对比了。这就意味着：原本匹配的，挪到后面就会变成不匹配（差异 $+1$）；原本不匹配的，挪到后面就会变成匹配（差异 $-1$）。

那么，只需一个简单的 $if$，就把滑动窗口内 `出窗口` + `进窗口` 两个操作合并成一步！

代码如下，已附加注释：

```Python
class Solution:
    def minFlips(self, s: str) -> int:
        n = len(s)
        # 基准交替模式
        target = '01'
        # 当前窗口不与 0101 不匹配的字符数
        diff_count = 0
        
        # 第一个窗口
        for i in range(n):
            if s[i] != target[i & 1]:
                diff_count += 1
        
        ans = min(diff_count, n - diff_count)
        # 特判偶数
        if n & 1 == 0:
            return ans
            
        # 开始滑动，把 s[i] 移出窗口
        for i in range(n):
            # n 是奇数，所以 i 和 i+n 的奇偶性必然相反
            # 同一个字符，原来在头部如果匹配，放到尾部就一定不匹配
            if s[i] == target[i & 1]:
                diff_count += 1
            else:
                diff_count -= 1
            
            # 更新
            ans = min(ans, diff_count, n - diff_count)
        
        return ans
```

```Java
// java
class Solution {
    public int minFlips(String s) {
        int n = s.length();
        // 基准交替模式
        String target = "01";
        // 当前窗口不与 0101 不匹配的字符数
        int diffCount = 0;
        
        // 第一个窗口
        for (int i = 0; i < n; i++) {
            if (s.charAt(i) != target.charAt(i & 1)) {
                diffCount += 1;
            }
        }
        
        int ans = Math.min(diffCount, n - diffCount);
        // 特判偶数
        if ((n & 1) == 0) {
            return ans;
        }
            
        // 开始滑动，把 s[i] 移出窗口
        for (int i = 0; i < n; i++) {
            // n 是奇数，所以 i 和 i+n 的奇偶性必然相反
            // 同一个字符，原来在头部如果匹配，放到尾部就一定不匹配
            if (s.charAt(i) == target.charAt(i & 1)) {
                diffCount += 1;
            } else {
                diffCount -= 1;
            }
            
            // 更新
            ans = Math.min(ans, Math.min(diffCount, n - diffCount));
        }
        
        return ans;
    }
}
```

- 时间复杂度： $O(n)$，至多两次遍历
- 空间复杂度： $O(1)$，仅使用常数个额外变量
