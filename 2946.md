[Problem: 2946. 循环移位后的矩阵相似检查](https://leetcode.cn/problems/matrix-similarity-after-cyclic-shifts/description/)

### 方法：模拟 & 优化

题意很清晰，注意本题下标是从 $0$ 开始，其中 $mat[0]$ 是偶数行。

循环移动 $k$ 次，如果 $k$ 大于列数 $n$，其实际效果等同于移动 $k \pmod n$ 次。例如，列长为 $4$，移动 $5$ 次和移动 $1$ 次的效果是一样的。

**模拟** 题意，遍历矩阵的每一行：

- 如果是 **偶数行**，构造它左移 $k$ 位后的新行，与原行比较。
- 如果是 **奇数行**，构造它右移 $k$ 位后的新行，与原行比较。

如果所有行都匹配，返回 $True$；只要有一行不匹配，立即返回 $False$。

代码如下，已附加注释：

```Python
# python
class Solution:
    def areSimilar(self, mat: List[List[int]], k: int) -> bool:
        m = len(mat)
        n = len(mat[0])
        
        # 简化位移次数
        k %= n
        # 矩阵不会有任何变化
        if k == 0:
            return True
        
        # 遍历每一行
        for i in range(m):
            row = mat[i]
            if i % 2 == 0:
                # 左移 k 位
                shifted_row = row[k:] + row[:k]
            else:
                # 右移 k 位
                shifted_row = row[n-k:] + row[:n-k]
            
            # 比较当前行是否一致
            if row != shifted_row:
                return False
        
        return True
```

```Java
// java
class Solution {
    public boolean areSimilar(int[][] mat, int k) {
        int m = mat.length;
        int n = mat[0].length;
        
        // 简化位移次数
        k %= n;
        // 矩阵不会有任何变化
        if (k == 0) {
            return true;
        }
        
        // 遍历每一行
        for (int i = 0; i < m; i++) {
            int[] row = mat[i];
            if (i % 2 == 0) {
                // 左移 k 位
                for (int j = 0; j < n; j++) {
                    if (row[j] != row[(j + k) % n]) {
                        return false;
                    }
                }
            } else {
                // 右移 k 位
                for (int j = 0; j < n; j++) {
                    if (row[j] != row[(j - k % n + n) % n]) {
                        return false;
                    }
                }
            }
        }
        
        return true;
    }
}
```

- 时间复杂度： $O(m\times n)$，其中 $m,n$ 是矩阵 $grid$ 的行数、列数
- 空间复杂度： $O(n)$，表示中间列表 $shifted$ 的大小

### 优化

对于任意一行，如果它在循环移动 $k$ 位后保持不变，这意味着该行具有以 $k$ 为步长的 **周期性**。举个例子，假设 $k=1, n=4$：

- **左移 $k$ 位**：原序列 $[a, b, c, d]$ 变成 $[b, c, d, a]$。要相等，必须 $a=b, b=c, c=d, d=a$。
- **右移 $k$ 位**：变为 $[d,a,b,c]$，虽然方向不同，但要使变换后数组不变，那么原数组在对应的偏移量上元素必须相等。

**核心结论**：不需要真的去区分左移还是右移。如果一行代码循环移动 $k$ 个位置后还能和原来一模一样，那么无论向左还是向右，它都必须满足 $row[i] = row[(i + k) % n]$。

不好理解？

在模运算中， $(j - k) \% n$ 实际上等同于 $(j + (n - k)) \% n$。

如果一个数组满足 $row[j] = row[(j + k) \% n]$ 对于所有的 $j$ 都成立，那么：我们可以把 $j$ 替换为 $j - k$，得到 $row[j - k] = row[(j - k + k) \% n]$  $\Rightarrow$  $row[j - k] = row[j]$。

这意味着：**“每一个元素等于它右边距离 $k$ 的元素”** 和 **“每一个元素等于它左边距离 $k$ 的元素”** 表达的是同一个逻辑约束——即数组具有以 $k$ 为步长的 **周期性**。

或者，想象把这一行数字排成一个**圆圈**：

- 循环右移 $k$ 次，就是把这个圆圈顺时针转了 $k$ 小段。
- 循环左移 $k$ 次，就是把这个圆圈逆时针转了 $k$ 小段。

如果这个圆圈在转动 $k$ 段后看起来和原来 **一模一样**，那么无论你是顺时针转还是逆时针转，它都必须满足相同的对称条件（即每隔 $k$ 个位置的数字必须相同）。

代码如下，已附加注释：

```Python
# python
class Solution:
    def areSimilar(self, mat: List[List[int]], k: int) -> bool:
        m = len(mat)
        n = len(mat[0])
        
        for i in range(m):
            for j in range(n):
                # 本质都是间隔 k
                if mat[i][j] != mat[i][(j + k) % n]:
                    return False
        
        return True
```

```Java
// java
class Solution {
    public boolean areSimilar(int[][] mat, int k) {
        int m = mat.length;
        int n = mat[0].length;
        
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                // 本质都是间隔 k
                if (mat[i][j] != mat[i][(j + k) % n]) {
                    return false;
                }
            }
        }
        
        return true;
    }
}
```

- 时间复杂度： $O(m\times n)$，一次遍历
- 空间复杂度： $O(1)$，仅使用常数个额外变量
