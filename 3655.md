[Problem: 3655. 区间乘法查询后的异或 II](https://leetcode.cn/problems/xor-after-range-multiplication-queries-ii/description/)

### 方法：根号分治

#### 寻找规律

在前一题 $3653$ 中，我们发现，直接暴力模拟的时间复杂度是 $O(q\times \frac{n}{k})$，在最坏情况下（当 $k=1$ 时）会退化到 $O(q\times n)$，在本题 $n, q = 10^5$ 的规模显然会超时。

分析一下暴力做法的问题：

- 当 $k$ 很大时（例如 $k > \sqrt{n}$）：更新的次数很少，最多只有 $\sqrt{n}$ 次。
- 当 $k$ 很小时（例如 $k=1, 2, 3$）：更新的次数非常多，接近 $n$ 次。

如果换个角度来看，

- 当 $k$ 大时，受影响元素是 **稀疏** 的（适合暴力）。
- 当 $k$ 小时，受影响元素是 **稠密** 且 **有周期性** 的（适合批量处理）。

因此，想到划分边界 $B$，将 “稀疏性” 和 “周期性” **各自** 的优势发挥到极致。也就是说，根据属性 $k$，将本题划分为两个子问题并分别采用最适算法的策略 —— **分治**！

---

#### 分治

我们将查询分为两类，设定一个阈值 $B$：

- **大步长情况 $k \ge B$**：直接暴力更新。
  - 每个查询最多更新 $n/B$ 个元素。
  - 总时间复杂度： $O(q \cdot \frac{n}{B})$。
- **小步长情况 $k < B$**：不能暴力，需要寻找规律。
  - 对于固定的 $k$，所有操作都在 $i \equiv \text{start} \pmod k$ 的链上进行。
  - 比如 $k=2$，它实际上把数组拆成了两部分：偶数下标链和奇数下标链。
  - 比如 $k=3$，拆成了三部分：取模为 $0$ 的链、取模为 $1$ 的链、取模为 $2$ 的链。
  - 在每一条链上，更新操作变成了 **普通的区间乘法**。

> 最通常的做法是取 $B=\sqrt{n}$，比较符合思维逻辑，也可以取其他数值。经实测，换成常数 $40$ 等速度会更快，一般想不到。

小步长怎么办？

为了方便操作，不是立即更新 $nums$。对于一个步长 $k$ 和余数 $rem \in [0, k-1]$，将属于该链的元素提取出来。原本的 “每隔 $k$ 个更新” 变成了这组新序列上的 “连续区间更新”。

举个例子，比如数组为 $nums=[1,2,3,4,5,6]$，查询是 $[0,3,2,9]$，那么这条链是 $[1,3,5]$，更新范围是 $[1,3]$，对这个范围内的数字全乘以 $9$。

---

#### 差分

如何高效处理区间乘法？

类比区间加法，想到使用 **乘法意义下的差分数组**。

设 $diff[k][rem][i]$ 表示步长为 $k$，余数为 $rem$ 的链上，第 $i$ 个位置累乘的值。

对于查询 $[l, r, k, v]$：

1. 计算 $rem = l \% k$。
2. 在该 $k$ 和 $rem$ 对应的差分结构中，在位置 $l$ 乘上 $v$，在位置 $r+k$ 除去 $v$（即乘上 $v$ 的逆元）。
3. 最后扫描一遍，通过前缀积还原。

注意，在区间加法中，普通的差分数组是 `diff[i] = a[i] - a[i-1]`，还原是 `sum(diff[0...i])`。而这里的差分是 **步长** 差分：即 $diff[i]$ 只影响 $i, i+k, i+2k...$。所以还原时是 `diff[i] *= diff[i-k]`。

如果你对普通差分的原理感兴趣，可以看这篇 [题解](https://leetcode.cn/problems/number-of-students-doing-homework-at-a-given-time/solutions/2899892/chai-fen-mei-ju-chai-fen-yuan-li-ying-yo-pbel/)。当然，如果你想进一步了解具体应用，可以看看 [Wiki百科](https://oi-wiki.org/basic/prefix-sum/#%E5%B7%AE%E5%88%86)。

乘法逆元怎么求？那就是数学问题 —— 费马小定理，感兴趣的小伙伴自行了解。

代码如下，已附加注释：

```Python
# python
class Solution:
    def xorAfterQueries(self, nums: List[int], queries: List[List[int]]) -> int:
        n = len(nums)
        MOD = 10**9 + 7
        # 阈值
        B = int(n**0.5) 
        # 小步长，按步长 k 分组存储
        small_k_queries = [[] for _ in range(B)]
        
        for l, r, k, v in queries:
            if v == 1:
                continue
            if k >= B:
                # 大步长，直接修改原数组
                for i in range(l, r + 1, k):
                    nums[i] = (nums[i] * v) % MOD
            else:
                # 小步长，暂时记录
                small_k_queries[k].append((l, r, v))
        
        # 统一处理
        for k in range(1, B):
            if not small_k_queries[k]:
                continue
            # 乘法差分数组
            diff = [1] * (n + k + 1)
            
            for l, r, v in small_k_queries[k]:
                # 终点 r 之后在同一步长序列中的下一个位置
                end_pos = l + ((r - l) // k + 1) * k
                
                # 起点乘 v
                diff[l] = (diff[l] * v) % MOD
                inv_v = pow(v, MOD - 2, MOD)  # 费马小定理求逆元
                # 终点后位置乘逆元
                if end_pos < n + k:
                    diff[end_pos] = (diff[end_pos] * inv_v) % MOD
            
            # 还原差分数组：每个点的值等于其对应序列的前缀积
            for i in range(n):
                if i >= k:
                    diff[i] = (diff[i] * diff[i-k]) % MOD
                # 应用到原数组
                nums[i] = (nums[i] * diff[i]) % MOD
        
        ans = 0
        for x in nums:
            ans ^= x
        return ans
```

```Java
// java
class Solution {
    public int xorAfterQueries(int[] nums, int[][] queries) {
        int n = nums.length;
        long MOD = 1_000_000_007L;
        // 阈值
        int B = (int) Math.sqrt(n); 
        // 小步长，按步长 k 分组存储
        List<int[]>[] smallKQueries = new ArrayList[B];
        for (int i = 0; i < B; i++) {
            smallKQueries[i] = new ArrayList<>();
        }
        
        for (int[] q : queries) {
            int l = q[0], r = q[1], k = q[2], v = q[3];
            if (v == 1) {
                continue;
            }
            if (k >= B) {
                // 大步长，直接修改原数组
                for (int i = l; i <= r; i += k) {
                    nums[i] = (int) (((long) nums[i] * v) % MOD);
                }
            } else {
                // 小步长，暂时记录
                smallKQueries[k].add(new int[]{l, r, v});
            }
        }
        
        // 统一处理
        for (int k = 1; k < B; k++) {
            if (smallKQueries[k].isEmpty()) {
                continue;
            }
            // 乘法差分数组
            long[] diff = new long[n + k + 1];
            Arrays.fill(diff, 1L);
            
            for (int[] sq : smallKQueries[k]) {
                int l = sq[0], r = sq[1], v = sq[2];
                // 终点 r 之后在同一步长序列中的下一个位置
                int endPos = l + ((r - l) / k + 1) * k;
                
                // 起点乘 v
                diff[l] = (diff[l] * v) % MOD;
                long invV = power(v, MOD - 2, MOD);  // 费马小定理求逆元
                // 终点后位置乘逆元
                if (endPos < n + k) {
                    diff[endPos] = (diff[endPos] * invV) % MOD;
                }
            }
            
            // 还原差分数组：每个点的值等于其对应序列的前缀积
            for (int i = 0; i < n; i++) {
                if (i >= k) {
                    diff[i] = (diff[i] * diff[i - k]) % MOD;
                }
                // 应用到原数组
                nums[i] = (int) (((long) nums[i] * diff[i]) % MOD);
            }
        }
        
        int ans = 0;
        for (int x : nums) {
            ans ^= x;
        }
        return ans;
    }

    // 快速幂计算逆元
    private long power(long base, long exp, long mod) {
        long res = 1;
        base %= mod;
        while (exp > 0) {
            if (exp % 2 == 1) res = (res * base) % mod;
            base = (base * base) % mod;
            exp /= 2;
        }
        return res;
    }
}
```

大步长部分：普通乘法，复杂度是 $O(q \cdot \sqrt{n})$。小步长部分：遍历 $k$ 从 1 到 $\sqrt{n}$，每个 $k$ 遍历一次 $n$，复杂是 $O(n \cdot \sqrt{n})$。

- 时间复杂度： $O((q+n)\sqrt{n})$，其中 $q$ 表示数组 $queries$ 的长度
- 空间复杂度： $O(q+n)$，表示列表 $smallKQueries$ 以及数组 $diff$ 的大小
