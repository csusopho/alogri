[Problem: 2452. 距离字典两次编辑以内的单词](https://leetcode.cn/problems/words-within-two-edits-of-dictionary/description/)

### 方法：暴力 & 字典树

题意很简单，编辑次数 = 不同字母个数。

**最暴力** 的思路：拿 $queries$ 中的每一个单词，去和 $dictionary$ 里的每一个单词硬碰硬地比对。

- 外层遍历 $queries$，内层遍历 $dictionary$。
- 因为题目说明了所有单词长度相同，我们可以直接用一个循环对比相同索引下的字符。

在对比两个单词时，一旦发现不同字符的数量 $diff$ 超过了 2，就立刻停止当前的对比，因为这个 $dictionary$ 单词已经不符合要求了。

只要在 $dictionary$ 中找到了 **任何一个** 满足条件的单词，就判定当前 $query$ 合法，直接跳到下一个 $query$。

代码如下，已附加注释：

```Python
# python
class Solution:
    def twoEditWords(self, queries: List[str], dictionary: List[str]) -> List[str]:
        res = []
        
        for word in queries:
            found = False
            # 检查与字典中每个单词的差异
            for dict_word in dictionary:
                diff = 0
                for a, b in zip(word, dict_word):
                    if a != b:
                        diff += 1
                        if diff > 2:  # 提前终止
                            break
                if diff <= 2:
                    found = True
                    break
            # 不超过两次
            if found:
                res.append(word)
        
        return res
```

```Java
// java
class Solution {
    public List<String> twoEditWords(String[] queries, String[] dictionary) {
        List<String> res = new ArrayList<>();
        
        for (String word : queries) {
            boolean found = false;
            // 检查与字典中每个单词的差异
            for (String dictWord : dictionary) {
                int diff = 0;
                for (int i = 0; i < word.length(); i++) {
                    if (word.charAt(i) != dictWord.charAt(i)) {
                        diff++;
                        if (diff > 2) {  // 提前终止
                            break;
                        }
                    }
                }
                if (diff <= 2) {
                    found = true;
                    break;
                }
            }
            // 不超过两次
            if (found) {
                res.add(word);
            }
        }
        
        return res;
    }
}
```

- 时间复杂度： $O(n\times m\times L)$，其中 $n,m$ 分别为数组 $queries,dictionary$ 的长度，而 $L$ 表示单词的平均长度
- 空间复杂度： $O(1)$，仅使用常数个额外变量

### 字典树

为什么想到 $Trie$？

如果 $dictionary$ 中有 $100$ 个单词都以 `trans...` 开头。在暴力法中，如果 $query$ 单词开头也是 `trans...`，那么会对这 $100$ 个单词分别对比前面的字符。而在 $Trie$ 中，这 $100$ 个单词的前缀在路径上是重合的，你只需走一次这条路径，极大地减少了冗余操作。

也即：利用前缀树压缩字典中的公共前缀，并利用 $DFS$ 进行状态分支搜索。

什么是字典树？可以看这篇 [图解](https://leetcode.cn/problems/minimum-number-of-valid-strings-to-form-target-i/solutions/3022452/trie-yuan-li-tui-dao-mo-ban-tu-jie-zi-di-w1sz/)，包含原理、模版、应用，对应的 $WiKi$ 百科为 [传送门](https://oi-wiki.org/string/trie/)。

**第一步：构建 Trie**。

将 $dictionary$ 中所有的单词插入 $Trie$，其中的每个节点代表一个字母。

**第二步：带容错的搜索**。

对于 $queries$ 中的每个单词 $word$，从 $Trie$ 的根节点开始递归搜索：

定义搜索函数 $search(node, index, edits)$。其中 $node$ 表示当前所在的 $Trie$ 节点，而 $index$ 表示当前匹配到 $word$ 的第几个字符，以及 $edits$ 表示已经使用了多少次编辑机会（上限为 $2$）。

- 不消耗编辑次数：如果 $word[index]$ 存在于当前节点的子节点中，顺着往下走，$edits$ 不变。
- 消耗编辑次数：**尝试替换。** 遍历当前节点的所有子节点，如果某个子节点字母不等于 $word[index]$，则消耗一次编辑机会，递归向下。

如果 $edits > 2$，此路不通，回溯。如果 $index = len(word)$，且当前 $Trie$ 节点标记为单词结尾，说明找到了匹配，返回 $True$。

代码如下，已附加注释：

```Python
# python
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_word = False

class Solution:
    def twoEditWords(self, queries, dictionary):
        # 构建字典树
        root = TrieNode()
        for word in dictionary:
            node = root
            for c in word:
                if c not in node.children:
                    node.children[c] = TrieNode()
                node = node.children[c]
            node.is_word = True
        
        res = []
        for word in queries:
            if self.search_with_edits(root, word, 0, 0):
                res.append(word)
        return res
    
    def search_with_edits(self, node, word, index, edits):
        if edits > 2:
            return False
        if index == len(word):
            return node.is_word
        
        c = word[index]
        # 尝试不编辑的情况
        if c in node.children:
            if self.search_with_edits(node.children[c], word, index+1, edits):
                return True
        
        # 尝试编辑的情况（替换为其他字符）
        for char in node.children:
            if char != c:
                if self.search_with_edits(node.children[char], word, index+1, edits+1):
                    return True
        
        return False
```

```Java
// java
class TrieNode {
    TrieNode[] children = new TrieNode[26];
    boolean isWord = false;
}

class Solution {
    public List<String> twoEditWords(String[] queries, String[] dictionary) {
        // 构建字典树
        TrieNode root = new TrieNode();
        for (String word : dictionary) {
            TrieNode node = root;
            for (char c : word.toCharArray()) {
                int idx = c - 'a';
                if (node.children[idx] == null) {
                    node.children[idx] = new TrieNode();
                }
                node = node.children[idx];
            }
            node.isWord = true;
        }
        
        List<String> res = new ArrayList<>();
        for (String word : queries) {
            if (searchWithEdits(root, word, 0, 0)) {
                res.add(word);
            }
        }
        return res;
    }
    
    private boolean searchWithEdits(TrieNode node, String word, int index, int edits) {
        if (edits > 2) {
            return false;
        }
        if (index == word.length()) {
            return node.isWord;
        }
        
        int targetIdx = word.charAt(index) - 'a';
        
        // 尝试不编辑的情况
        if (node.children[targetIdx] != null) {
            if (searchWithEdits(node.children[targetIdx], word, index + 1, edits)) {
                return true;
            }
        }
        
        // 尝试编辑的情况（替换为其他字符）
        for (int i = 0; i < 26; i++) {
            if (i != targetIdx && node.children[i] != null) {
                if (searchWithEdits(node.children[i], word, index + 1, edits + 1)) {
                    return true;
                }
            }
        }
        
        return false;
    }
}
```

- 时间复杂度： $O(m\times L+n\times L^2)$，对于一个长度为 $L$ 的单词，允许 $k$ 次错误，一个单词查询的理论最坏时间复杂度是 $O(L^k \cdot \Sigma^k)$
- 空间复杂度： $O(M\times L\times |\Sigma|)$，其中 $|\Sigma|$ 是字符集大小
