[Problem: 2033. 获取单值网格的最小操作数](https://leetcode.cn/problems/minimum-operations-to-make-a-uni-value-grid/description/)

### 方法：贪心

把所有数字全变为相同，比如 $[1,2,3]$ 最终变为 $[2,2,2]$。

一个比较容易想到的思路：最终全变成 **中位数**。

- 比如 $[1,2,2,2,3]$，最少的操作一定是修改占比较小的 $[1,3]$；
- 比如 $[1,4,6,7,9]$，最少的操作应该是向中间靠拢，全变为 $6$。

怎么想到贪心？

假设最终所有元素都变为目标值 $T$。对于网格中的每一个数 $a_i$，将其变为 $T$ 所需的操作数是 $\frac{|a_i - T|}{x}$。总操作数就是：

$$
S = \sum \frac{|a_i - T|}{x} = \frac{1}{x} \sum |a_i - T|
$$

因为 $x$ 是常数，最小化 $S$ 等价于最小化 $\sum |a_i - T|$。

**物理直觉**：如果在一根数轴上放置了若干个点，想找一个位置 $T$，使得所有点到 $T$ 的距离之和最短。直觉告诉我们，这个 $T$ 应该在这些点的 “中间”。

为什么贪心思路正确？

假设当前的候选目标值是 $T$。将数组中 **小于** $T$ 的元素个数，记为 $count_{small}$。而数组中 **大于** $T$ 的元素个数，记为 $count_{big}$。

如果我们把 $T$ 稍微向右移动一个单位距离 $\Delta$：

- 所有小于 $T$ 的点到 $T$ 的距离都会 **增加** $\Delta$。总增加： $count_{small} \cdot \Delta$。
- 所有大于 $T$ 的点到 $T$ 的距离都会 **减少** $\Delta$。总减少： $count_{big} \cdot \Delta$。

因此，得出结论：

- 如果 $count_{big} > count_{small}$，向右移动 $T$ 会使总距离减小。
- 如果 $count_{small} > count_{big}$，向左移动 $T$ 会使总距离减小。
- 只有当 $count_{small} = count_{big}$ 时（即 $T$ 处于中位数位置），总距离达到最小，无法再通过移动来优化。

注意，本题要求所有数字只能 $\pm x$，所以所有数必须 **同余**，也即 `a % x == b % x`，否则无法最终变为相同数字，必须提前检查！

由于数组元素是离散的，

- 当元素个数为 **奇数** 时，中位数是唯一的，就是排序后中间的那个数。
- 当元素个数为 **偶数** 时，排序后中间的两个数以及它们之间的任意满足 `mod x` 条件的数都可以作为 $T$，它们产生的总距离是相等的。为了方便，直接取 $nums[n // 2]$。

总的来看，我们只需要将 $nums$ 数字变为一维，然后排序求中位数，遍历计算差值之和。

代码如下，已附加注释：

```Python
# python
class Solution:
    def minOperations(self, grid: List[List[int]], x: int) -> int:
        # 扁平化
        nums = []
        for row in grid:
            nums.extend(row)
        
        # 检查余数合法性
        base_mod = nums[0] % x
        for num in nums:
            if num % x != base_mod:
                return -1
        
        # 排序，找到中位数
        nums.sort()
        n = len(nums)
        median = nums[n // 2]
        
        # 计算总操作步数
        ans = 0
        for num in nums:
            ans += abs(num - median) // x
        
        return ans
```

```Java
// java
class Solution {
    public int minOperations(int[][] grid, int x) {
        int m = grid.length;
        int n = grid[0].length;
        
        // 扁平化
        int[] nums = new int[m * n];
        int idx = 0;
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                nums[idx++] = grid[i][j];
            }
        }
        
        // 检查余数合法性
        int baseMod = nums[0] % x;
        for (int num : nums) {
            // 避免负数情况
            if ((num - nums[0]) % x != 0) {
                return -1;
            }
        }
        
        // 排序，找到中位数
        Arrays.sort(nums);
        int total = m * n;
        int median = nums[total / 2];
        
        // 计算总操作步数
        int ans = 0;
        for (int num : nums) {
            ans += Math.abs(num - median) / x;
        }
        
        return ans;
    }
}
```

- 时间复杂度： $O(mnlog(mn))$，其中 $m,n$ 是数组 $grid$ 的行/列数，时间瓶颈在于排序
- 空间复杂度： $O(mn)$，表示一维 $nums$ 的大小，排序的栈开销为 $O(log(mn))$
