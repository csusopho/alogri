[Problem: 1878. 矩阵中最大的三个菱形和](https://leetcode.cn/problems/get-biggest-three-rhombus-sums-in-a-grid/description/)

### 方法：暴力 & 前缀和

仔细观察，发现一个菱形由其 **中心点** $(r, c)$ 和 **半径** $k$ 决定。

- 当 $k=0$ 时，菱形是一个点，总和为 $grid[r][c]$。
- 当 $k>0$ 时，菱形有四个顶点：
  - 上顶点： $(r - k, c)$
  - 下顶点： $(r + k, c)$
  - 左顶点： $(r, c - k)$
  - 右顶点： $(r, c + k)$

**暴力** 想法：枚举矩阵中的每一个格子作为菱形的中心，并尝试所有可能的半径 $k$。

具体实现时，可以枚举中心，或者枚举某个顶点。这里选择枚举 $(r,c)$ 作为菱形的 **上顶点**，需要判断其它三个顶点（下、左、右）是否超出矩阵范围。

- 下顶点：$(r + 2k, c)$
- 左顶点：$(r + k, c - k)$
- 右顶点：$(r + k, c + k)$

**流程**：遍历每一个格子 $(r, c)$ 作为上顶点，半径 $k = 0, 1, 2, \dots$ 直到顶点超出边界。对于每个有效的 $(r, c, k)$，使用四个 $for$ 循环分别遍历菱形的四条边，累加边界上的元素。

此外，使用一个集合 $set$ 存储所有计算出的和，以保证 “互不相同”。最后对集合排序，取前三个最大值。

代码如下，已附加注释：

```Python
# python
class Solution:
    def getBiggestThree(self, grid: List[List[int]]) -> List[int]:
        m, n = len(grid), len(grid[0])
        res_set = set()

        for r in range(m):
            for c in range(n):
                # 每一个格子本身
                res_set.add(grid[r][c])
                
                # 枚举半径
                for k in range(1, m):
                    # 检查当前半径 k 下，菱形的四个顶点是否越界
                    if r + 2 * k >= m or c - k < 0 or c + k >= n:
                        break
                    
                    # 边界元素和
                    cur_sum = 0
                    # 右上边
                    for i in range(k):
                        cur_sum += grid[r + i][c + i]
                    # 右下边
                    for i in range(k):
                        cur_sum += grid[r + k + i][c + k - i]
                    # 左下边
                    for i in range(k):
                        cur_sum += grid[r + 2*k - i][c - i]
                    # 左上边
                    for i in range(k):
                        cur_sum += grid[r + k - i][c - k + i]
                    
                    res_set.add(cur_sum)

        # 降序排序
        sorted_sums = sorted(list(res_set), reverse=True)
        return sorted_sums[:3]
```

```Java
// java
class Solution {
    public int[] getBiggestThree(int[][] grid) {
        int m = grid.length, n = grid[0].length;
        Set<Integer> resSet = new HashSet<>();

        for (int r = 0; r < m; r++) {
            for (int c = 0; c < n; c++) {
                // 每一个格子本身
                resSet.add(grid[r][c]);
                
                // 枚举半径
                for (int k = 1; k < m; k++) {
                    // 检查当前半径 k 下，菱形的四个顶点是否越界
                    if (r + 2 * k >= m || c - k < 0 || c + k >= n) {
                        break;
                    }
                    
                    // 边界元素和
                    int curSum = 0;
                    // 右上边
                    for (int i = 0; i < k; i++) {
                        curSum += grid[r + i][c + i];
                    }
                    // 右下边
                    for (int i = 0; i < k; i++) {
                        curSum += grid[r + k + i][c + k - i];
                    }
                    // 左下边
                    for (int i = 0; i < k; i++) {
                        curSum += grid[r + 2 * k - i][c - i];
                    }
                    // 左上边
                    for (int i = 0; i < k; i++) {
                        curSum += grid[r + k - i][c - k + i];
                    }
                    
                    resSet.add(curSum);
                }
            }
        }

        // 降序排序
        List<Integer> sortedSums = new ArrayList<>(resSet);
        Collections.sort(sortedSums, Collections.reverseOrder());
        
        int size = Math.min(3, sortedSums.size());
        int[] result = new int[size];
        for (int i = 0; i < size; i++) {
            result[i] = sortedSums.get(i);
        }
        return result;
    }
}
```

- 时间复杂度： $O(m \cdot n \cdot \min(m, n)^2)$，前两层循环是 $O(m \cdot n)$，后两层每一次循环是 $O(\min(m, n))$
- 空间复杂度： $O(m \cdot n)$，主要消耗在存储菱形和的 $set$，最坏情况下不超过 $O(m \cdot n \cdot \min(m, n))$

### 前缀和

枚举边长累加太慢，如何快速计算？

这种 *子数组求和* 问题，经典做法是 **前缀和**。具体原理/做法，可以看看这篇 [题解](https://leetcode.cn/problems/minimum-operations-to-make-all-array-elements-equal/solutions/2842529/pai-xu-qian-zhui-he-er-fen-fei-chang-qin-biux/)。在本题中，菱形的边是倾斜 $45$ 度的，这启发我们使用对角线前缀和。

维护两个前缀和矩阵：

- **$d1$ (主对角线方向 $\searrow$)**：从左上到右下。定义 $d1[i][j]$ 表示从该对角线起点到 $grid[i][j]$ 的累加和。公式为 $d1[i][j] = grid[i][j] + d1[i-1][j-1]$。
- **$d2$ (副对角线方向 $\swarrow$)**：从右上到左下。定义 $d2[i][j]$ 表示从该对角线起点到 $grid[i][j]$ 的累加和。公式为 $d2[i][j] = grid[i][j] + d2[i-1][j+1]$。

假设一个菱形的四个顶点为：

- $Top:(r, c)$
- $Right:(r+k, c+k)$
- $Bottom:(r+2k, c)$
- $Left:(r+k, c-k)$

这四条边的和可以表示为：

- **右上边**：属于 $d1$ 方向，总和为 $d1[r+k][c+k] - d1[r-1][c-1]$
- **左下边**：属于 $d1$ 方向，总和为 $d1[r+2k][c] - d1[r+k-1][c-k-1]$
- **左上边**：属于 $d2$ 方向，总和为 $d2[r+k][c-k] - d2[r-1][c+1]$
- **右下边**：属于 $d2$ 方向，总和为 $d2[r+2k][c] - d2[r+k-1][c+k+1]$

注意，这四条线段相加时，四个顶点都被 **重复计算** 了一次，因此最后的结果需要减去这四个顶点的值。

处理边界时，需要补零来处理 $i-1$ 的情况，记得在坐标对应 $d$ 矩阵需要 $+1$。

代码如下，已附加注释：

```Python
# python
class Solution:
    def getBiggestThree(self, grid: List[List[int]]) -> List[int]:
        m, n = len(grid), len(grid[0])
        # 预处理对角线前缀和
        # d1: ↘ 方向
        # d2: ↙ 方向
        d1 = [[0] * (n + 2) for _ in range(m + 2)]
        d2 = [[0] * (n + 2) for _ in range(m + 2)]
        
        for r in range(m):
            for c in range(n):
                d1[r+1][c+1] = grid[r][c] + d1[r][c]
                d2[r+1][c+1] = grid[r][c] + d2[r][c+2]
        
        sums = set()
        
        # 枚举上顶点
        for r in range(m):
            for c in range(n):
                # 每一个格子本身
                sums.add(grid[r][c])
                
                # 枚举半径
                for k in range(1, m):
                    # 检查四个顶点是否越界
                    if r + 2 * k >= m or c - k < 0 or c + k >= n:
                        break
                    
                    # 四个顶点的坐标
                    top = (r, c)
                    bottom = (r + 2*k, c)
                    left = (r + k, c - k)
                    right = (r + k, c + k)
                    
                    # 右上边↘
                    s1 = d1[right[0]+1][right[1]+1] - d1[top[0]][top[1]]
                    # 左下边↘
                    s2 = d1[bottom[0]+1][bottom[1]+1] - d1[left[0]][left[1]]
                    # 左上边↙
                    s3 = d2[left[0]+1][left[1]+1] - d2[top[0]][top[1]+2]
                    # 右下边↙
                    s4 = d2[bottom[0]+1][bottom[1]+1] - d2[right[0]][right[1]+2]
                    
                    # 总和
                    cur_sum = s1 + s2 + s3 + s4 - grid[top[0]][top[1]] - grid[bottom[0]][bottom[1]] - grid[left[0]][left[1]] - grid[right[0]][right[1]]
                    sums.add(cur_sum)
        
        # 排序
        return sorted(list(sums), reverse=True)[:3]
```

```Java
// java
class Solution {
    public int[] getBiggestThree(int[][] grid) {
        int m = grid.length, n = grid[0].length;
        // 预处理对角线前缀和
        // d1: ↘ 方向
        // d2: ↙ 方向
        int[][] d1 = new int[m + 2][n + 2];
        int[][] d2 = new int[m + 2][n + 2];
        
        for (int r = 0; r < m; r++) {
            for (int c = 0; c < n; c++) {
                d1[r + 1][c + 1] = grid[r][c] + d1[r][c];
                d2[r + 1][c + 1] = grid[r][c] + d2[r][c + 2];
            }
        }
        
        Set<Integer> sums = new HashSet<>();
        
        // 枚举上顶点
        for (int r = 0; r < m; r++) {
            for (int c = 0; c < n; c++) {
                // 每一个格子本身
                sums.add(grid[r][c]);
                
                // 枚举半径
                for (int k = 1; k < m; k++) {
                    // 检查四个顶点是否越界
                    if (r + 2 * k >= m || c - k < 0 || c + k >= n) {
                        break;
                    }
                    
                    // 四个顶点的坐标
                    int[] top = {r, c};
                    int[] bottom = {r + 2 * k, c};
                    int[] left = {r + k, c - k};
                    int[] right = {r + k, c + k};
                    
                    // 右上边↘
                    int s1 = d1[right[0] + 1][right[1] + 1] - d1[top[0]][top[1]];
                    // 左下边↘
                    int s2 = d1[bottom[0] + 1][bottom[1] + 1] - d1[left[0]][left[1]];
                    // 左上边↙
                    int s3 = d2[left[0] + 1][left[1] + 1] - d2[top[0]][top[1] + 2];
                    // 右下边↙
                    int s4 = d2[bottom[0] + 1][bottom[1] + 1] - d2[right[0]][right[1] + 2];
                    
                    // 总和
                    int curSum = s1 + s2 + s3 + s4 - grid[top[0]][top[1]] - grid[bottom[0]][bottom[1]] - grid[left[0]][left[1]] - grid[right[0]][right[1]];
                    sums.add(curSum);
                }
            }
        }
        
        // 排序
        List<Integer> sortedSums = new ArrayList<>(sums);
        Collections.sort(sortedSums, Collections.reverseOrder());
        
        int size = Math.min(3, sortedSums.size());
        int[] result = new int[size];
        for (int i = 0; i < size; i++) {
            result[i] = sortedSums.get(i);
        }
        return result;
    }
}
```

- 时间复杂度： $O(m \cdot n \cdot \min(m, n))$，将三者累乘记作 $L$，严谨地来看，应该考虑排序，复杂度变为 $O(L \cdot logL)$
- 空间复杂度： $O(m \cdot n \cdot \min(m, n))$，表示哈希表 $set$ 的大小

如果我们使用 **小顶堆** 来只维护最大的三个元素，而不是对所有结果进行全排序，那么排序部分的开销可以降到极低，感兴趣的小伙伴可以自行尝试。
