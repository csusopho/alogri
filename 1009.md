[Problem: 1009. 十进制整数的反码](https://leetcode.cn/problems/complement-of-base-10-integer/description/)

### 方法：模拟 & 异或

简单来说，反码就是 **反转** 二进制的每一位。

因此，我们直接 **模拟**：将整数 $n$ 转为二进制字符串，然后遍历每一位进行反转，最后再转回整数。

代码如下，已附加注释：

```Python
# python
class Solution:
    def bitwiseComplement(self, n: int) -> int:
        # 转为二进制字符串
        bin_str = bin(n)[2:]
        
        # 翻转每一位
        res_str = ""
        for char in bin_str:
            if char == '1':
                res_str += '0'
            else:
                res_str += '1'
         
        # 转回十进制
        return int(res_str, 2)
```

```Java
// java
class Solution {
    public int bitwiseComplement(int n) {
        // 转为二进制字符串
        String binStr = Integer.toBinaryString(n);
        
        // 翻转每一位
        StringBuilder resStr = new StringBuilder();
        for (char c : binStr.toCharArray()) {
            if (c == '1') {
                resStr.append('0');
            } else {
                resStr.append('1');
            }
        }
         
        // 转回十进制
        return Integer.parseInt(resStr.toString(), 2);
    }
}
```

- 时间复杂度： $O(L)$，其中 $L=log_{2}n$ 表示 $n$ 的二进制长度
- 空间复杂度： $O(L)$，表示字符串 $res$ 的长度

### 异或

在计算机底层，翻转每一位的本质是 **异或运算**。规律为：`0 ^ 1 = 1`, `1 ^ 1 = 0`。

所以，只需要构造一个和 $n$ 长度相同的、全为 $1$ 的二进制数 —— 称为 **掩码 Mask**，让 $n$ 与它异或即可得到结果。

以 $n = 5(101_2)$ 为例：

1. 确定位数：数字 $5$ 的二进制是 $101$，长度为 $3$。
2. 构造掩码：需要一个长度为 $3$ 的全 1 数，即 $111_2$，十进制 $7$。构造方法是 $2^3 - 1$ = $8 - 1$ = $7$。
3. 执行异或： $101_2 \oplus 111_2 = 010_2$，十进制为 $2$。

因此，对于任何 $n > 0$，掩码是大于 $n$ 的最小的 $2^n - 1$。特别 **注意**，如果 $n=0$，这个公式就无法成立，手动返回结果。

代码如下，已附加注释：

```Python
# python
class Solution:
    def bitwiseComplement(self, n: int) -> int:
        # 特殊处理
        if n == 0:
            return 1

        # 二进制长度
        length = n.bit_length()
        # 构造全 1 掩码
        mask = (1 << length) - 1
        
        # 异或
        return n ^ mask
```

```Java
// java
class Solution {
    public int bitwiseComplement(int n) {
        // 特殊处理
        if (n == 0) {
            return 1;
        }

        // 二进制长度
        int length = Integer.toBinaryString(n).length();
        // 构造全 1 掩码
        int mask = (1 << length) - 1;
        
        // 异或
        return n ^ mask;
    }
}
```

如果不想调库 $bit$ 计算二进制长度，可以通过 **位运算**，不断地向右填充 $1$ 来构造 $mask$。

1. 假设 $n = 101_2$
2. `n = (n << 1) | 1` -> 变成 $1011_2$
3. `n = (n << 1) | 1` -> 变成 $10111_2$

这种方法是逐位构造 $mask$，这样就无需担心 $n=0$ 的情况。

代码如下，已附加注释：

```Python
# python
class Solution:
    def bitwiseComplement(self, n: int) -> int:
        mask = 1
        
        # 逐位构造
        while mask < n:
            mask = (mask << 1) | 1
        
        # 异或
        return n ^ mask
```

```Java
// java
class Solution {
    public int bitwiseComplement(int n) {
        int mask = 1;
        
        // 逐位构造
        while (mask < n) {
            mask = (mask << 1) | 1;
        }
        
        // 异或
        return n ^ mask;
    }
}
```

- 时间复杂度： $O(L)$
- 空间复杂度： $O(1)$

为什么不直接使用 **取反** 运算符 `~`？

在 $Python$ 中，它是针对 **有符号整数** 的！

- `~5` 实际上会把所有的前导零也取反。
- 计算机中 $5$ 表示为 `...00000101`。
- `~5` 会变成 `...11111010`，结果是 $-6$。

题目要求只翻转 **到最高位为止**，所以我们必须手动构造一个长度匹配的掩码。
