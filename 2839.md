[Problem: 2839. 判断通过操作能否让字符串相等 I](https://leetcode.cn/problems/check-if-strings-can-be-made-equal-with-operations-i/description/)

### 方法：模拟

什么是下标满足 $j - i = 2$ 的字符？总共只有 $4$ 个字符：

- 下标为 **0** 的字符只能和下标为 **2** 的字符交换。
- 下标为 **1** 的字符只能和下标为 **3** 的字符交换。

换句话说，**偶数** 下标位置的字符集是固定的，**奇数** 下标位置的字符集也是固定的。

对于每一组（比如偶数组），因为一组的长度为 $2$，要使 $s1$ 的字符能通过交换变成 $s2$ 的字符，只有两种可能：

1. 对应的位置直接相等，即 $s1[0]=s2[0]$ 且 $s1[2]=s2[2]$。
2. 对应的位置交叉相等，即 $s1[0]=s2[2]$ 且 $s1[2]=s2[0]$。

如果字符串长度为 $6$，一组的长度为 $3$ 或者更大怎么办？那就使用 **排序**，让小/大的字符与小/大的字符匹配。速度更快的做法，就是使用 **哈希表** 统计每一组的频率，然后比较。毕竟字母最多 $26$ 个，比较的时间就会很短。

代码如下，已附加注释：

```Python
# python
class Solution:
    def canBeEqual(self, s1: str, s2: str) -> bool:
        # 偶数下标
        even_match = (s1[0] == s2[0] and s1[2] == s2[2]) or \
                    (s1[0] == s2[2] and s1[2] == s2[0])
        
        # 奇数下标
        odd_match = (s1[1] == s2[1] and s1[3] == s2[3]) or \
                    (s1[1] == s2[3] and s1[3] == s2[1])
        
        return even_match and odd_match
```

```Java
// java
class Solution {
    public boolean canBeEqual(String s1, String s2) {
        // 偶数下标
        boolean evenMatch = (s1.charAt(0) == s2.charAt(0) && s1.charAt(2) == s2.charAt(2)) ||
                            (s1.charAt(0) == s2.charAt(2) && s1.charAt(2) == s2.charAt(0));
        
        // 奇数下标
        boolean oddMatch = (s1.charAt(1) == s2.charAt(1) && s1.charAt(3) == s2.charAt(3)) ||
                           (s1.charAt(1) == s2.charAt(3) && s1.charAt(3) == s2.charAt(1));
        
        return evenMatch && oddMatch;
    }
}
```

- 时间复杂度： $O(1)$，进行常数次比较
- 空间复杂度： $O(1)$，使用常数个额外变量
