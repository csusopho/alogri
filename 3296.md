[Problem: 3296. 移山所需的最少秒数](https://leetcode.cn/problems/minimum-number-of-seconds-to-make-mountain-height-zero/description/)

### 方法：最小堆 & 二分

假设工人 $i$ 花费 $w[i]+w[i]\times 2$ 秒，**换个角度看**，就是一共工作了两次，每次都是降低 $1$ 个高度。每次工作后，下一次工作花费的时间增加！

那么，不必纠结于每个人到底干几次，只需要关心现在让谁干！—— 也即工人 $i$ 已工作完毕，增加了一个新的工人，他的工作花费为 $w[i]\times 2$。

**问题转化**：每次降低 $1$ 个高度，分给任意一个工人，直至高度变为 $0$。

**贪心**：为了让总时间最少，每次降低时，都交给那个 “当前能最快完成” 的工人。

每次如何获取 $n$ 个工人中的最小值？排序/有序结构。这里使用 **最小堆**。

注意，如果工人 $i$ 已经工作了 $x$ 次，下一次工作所需要的 **增量时间** 是：

$$
\text{cost} = \text{workerTimes}[i] \times (x + 1)
$$

也就是说，一个工人的工作时间，取决于已完成几次、基础时间。

因此，**堆中存储** `(下一次完成的总时刻, 当前已工作次数, 工人的基础时间)`。

初始时，将所有工人放入堆中，此时每个工人的 “完成时间” 就是 $workerTimes[i]$。

主循环 $mountainHeight$ 次：

- 从堆顶弹出 “总时刻” 最小的工人。
- 该工人负责将山的高度降低 $1$。
- 更新该工人的状态：
  - 已降低高度 $x \leftarrow x + 1$
  - 下一次完成总时刻 $\leftarrow \text{当前总时刻} + \text{workerTimes}[i] \times (x + 1)$
- 将更新后的工人信息重新压入堆。

最后一次弹出的 “总时刻” 即为答案。

本题工人是 **同时工作**，工人 $1,2$ 在第 $t$ 秒可以一起工作，互不干扰。这种解法怎么像是分开工作？注意，答案 $ans$ 记录的是第一维 $总时刻$ 的最大值。不管怎么算，一定是某个工人最后完成工作，他的完成时间就是答案。

代码如下，已附加注释：

```Python
# python
class Solution:
    def minNumberOfSeconds(self, mountainHeight: int, workerTimes: List[int]) -> int:
        # (下一次完成任务的时间点, 当前已完成的高度x, 工人的基础时间w)
        pq = [(w, 1, w) for w in workerTimes]
        heapq.heapify(pq)
        ans = 0

        for _ in range(mountainHeight):
            finish_time, x, w = heapq.heappop(pq)
            ans = finish_time  # 记录最后完成时间
            
            # 这个工人准备接下一个任务
            next_x = x + 1
            next_finish_time = w * next_x * (next_x + 1) // 2
            heapq.heappush(pq, (next_finish_time, next_x, w))
         
        return ans
```

```Java
// java
class Solution {
    public long minNumberOfSeconds(int mountainHeight, int[] workerTimes) {
        // (下一次完成任务的时间点, 当前已完成的高度x, 工人的基础时间w)
        PriorityQueue<long[]> pq = new PriorityQueue<>((a, b) -> Long.compare(a[0], b[0]));
        for (int w : workerTimes) {
            pq.offer(new long[]{(long) w, 1, (long) w});
        }
        long ans = 0;

        for (int i = 0; i < mountainHeight; i++) {
            long[] curr = pq.poll();
            long finish_time = curr[0];
            long x = curr[1];
            long w = curr[2];
            ans = finish_time;  // 记录最后完成时间
            
            // 这个工人准备接下一个任务
            long next_x = x + 1;
            long next_finish_time = w * next_x * (next_x + 1) / 2;
            pq.offer(new long[]{next_finish_time, next_x, w});
        }
         
        return ans;
    }
}
```

将山的高度 $mountainHeight$ 简记为 $h$，数组 $workTimes$ 的长度记作 $n$。

- 时间复杂度： $O(h\times logn)$，每次堆操作时间为 $O(logn)$
- 空间复杂度： $O(n)$，存储所有工人

### 二分

如果 $T$ 秒内能把山削平，那么 $T+1$ 秒肯定也能 —— **单调性**，因此想到二分。

左边界设为 $0$。右边界假设由最快的工人独自完成。若最快工人时间为 $w$，山高度为 $h$，如果工作全部交给他，则 $right = w \cdot \frac{h(h+1)}{2}$。

假设规定时间为 $T$ 秒，所有工人最多 **一共** 能降低多少高度？也就是说，每个工人都能工作 $T$ 秒，山的高度能降为 $0$ 吗？

- 对于工人 $i$，假设他在 $T$ 秒内降低了 $x_i$ 个高度，需满足：

$$
\text{workerTimes}[i] \cdot (1 + 2 + \dots + x_i) \le T
$$

也即

$$
\text{workerTimes}[i] \cdot \frac{x_i(x_i + 1)}{2} \le T
$$

- 我们需要算出每个工人最大的 $x_i$，如果 $\sum x_i \ge \text{mountainHeight}$，则说明 $T$ 秒可行。

如何快速计算 $x_i$？最简单的措施，利用 **求根公式** 解一元二次方程，也即

$$
x_i = \left\lfloor \frac{-1 + \sqrt{1 + \frac{8T}{\text{workerTimes}[i]}}}{2} \right\rfloor
$$

总结，检查 $check$ 函数的 **逻辑为**：遍历每个工人时长 $w$，通过公式计算该工人贡献的高度 $x$，累加到总高度 $total$ 中。如果 $total$ 最终 $\geq$ 总高度 $h$，说明成功，反之失败。

使用最容易理解的闭区间二分，代码如下，已附加注释：

```Python
# python
class Solution:
    def minNumberOfSeconds(self, mountainHeight: int, workerTimes: List[int]) -> int:
        # 判断能否在 limit 时间内完成任务
        def check(limit):
            total_h = 0
            for w in workerTimes:
                # 由公式反解 x
                x = int((math.sqrt(1 + 8 * limit / w) - 1) / 2)
                total_h += x

                if total_h >= mountainHeight:  # 提前退出
                    return True
            
            return total_h >= mountainHeight

        low = 0
        # 假设由最快的工人一个人完成
        min_w = min(workerTimes)
        high = min_w * mountainHeight * (mountainHeight + 1) // 2
        ans = high

        # 二分
        while low <= high:
            mid = (low + high) // 2
            if check(mid):
                ans = mid      # 当前时间可行，尝试更短时间
                high = mid - 1
            else:
                low = mid + 1  # 时间不够，增加时间
        return ans
```

```Java
// java
class Solution {
    public long minNumberOfSeconds(int mountainHeight, int[] workerTimes) {
        // 判断能否在 limit 时间内完成任务
        long low = 0;
        // 假设由最快的工人一个人完成
        int minW = workerTimes[0];
        for (int w : workerTimes) {
            if (w < minW) minW = w;
        }
        long high = (long) minW * mountainHeight * (mountainHeight + 1) / 2;
        long ans = high;

        // 二分
        while (low <= high) {
            long mid = low + (high - low) / 2;
            if (check(mid, mountainHeight, workerTimes)) {
                ans = mid;      // 当前时间可行，尝试更短时间
                high = mid - 1;
            } else {
                low = mid + 1;  // 时间不够，增加时间
            }
        }
        return ans;
    }

    private boolean check(long limit, int mountainHeight, int[] workerTimes) {
        long total_h = 0;
        for (int w : workerTimes) {
            // 由公式反解 x
            long x = (long) ((Math.sqrt(1 + 8.0 * limit / w) - 1) / 2);
            total_h += x;

            if (total_h >= mountainHeight) {  // 提前退出
                return true;
            }
        }
        return total_h >= mountainHeight;
    }
}
```

- 时间复杂度： $O(n\times logM)$，假设最快工人时长为 $w$，山高度为 $h$，则 $M=w \cdot \frac{h(h+1)}{2}$ 表示总二分次数，每次二分时间为 $O(n)$
- 空间复杂度： $O(1)$，仅使用常数个额外变量
