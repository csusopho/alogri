[Problem: 1391. 检查网格中是否存在有效路径](https://leetcode.cn/problems/check-if-there-is-a-valid-path-in-a-grid/description/)

### 方法：DFS

网格图，寻找从左上角 $(0, 0)$ 到右下角 $(m-1, n-1)$ 的有效路径 —— $DFS$。

首先，理解一下 “街道连接” 的本质。一个有效的移动必须满足两个条件：

1. **出发点**：当前单元格的街道类型支持向那个方向移动。
2. **目的地**：目标单元格的街道类型支持从相反的方向进入。

例如，如果从当前格子向 **右** 走，目标格子必须有一个向 **左** 的开口。

为了简化代码，将 $6$ 种街道类型转换为坐标偏移量：

- $1$: 左 $(0, -1)$，右 $(0, 1)$
- $2$: 上 $(-1, 0)$，下 $(1, 0)$
- $3$: 左 $(0, -1)$，下 $(1, 0)$
- $4$: 右 $(0, 1)$，下 $(1, 0)$
- $5$: 左 $(0, -1)$，上 $(-1, 0)$
- $6$: 右 $(0, 1)$，上 $(-1, 0)$

假设当前在 $(r, c)$，街道类型是 $grid[r][c]$，它允许移动的方向是 $(dr, dc)$。我们要移动到新位置 $(nr, nc)$，其中 $nr = r + dr, nc = c + dc$。

**连通的充要条件：** $(nr, nc)$ 处的街道必须包含指向回 $(r, c)$ 的方向，即包含 $(-dr, -dc)$。

接下来就是 $dfs$ 的经典操作：

创建一个 $visited$ 集合记录已访问的节点。

**定义** $dfs(r, c)$ 表示从 $(r,c)$ 出发，是否存在一条到达终点的路径。

边界情况：如果 $(r, c)$ 是右下角 $(m-1, n-1)$，返回 $True$。

对于当前节点，将 $(r, c)$ 加入 $visited$，表示已访问。接着，根据 $grid[r][c]$ 查找它支持的两个方向。

- 计算新坐标 $(nr, nc)$。
- 检查 $(nr, nc)$ 是否越界。
- 检查 $(nr, nc)$ 是否已访问。
- 检查 $grid[nr][nc]$ 是否有开口 **接回** $(r, c)$。
- 如果任一方向能到达终点，返回 $True$。

递归入口：从 $(0, 0)$ 开始调用。

代码如下，已附加注释：

```Python
# python
class Solution:
    def containsCycle(self, grid: List[List[str]]) -> bool:
        m, n = len(grid), len(grid[0])
        visited = [[False] * n for _ in range(m)]
```

```Java
// java
class Solution {
    private int[][][] directions = {
        {},
        {{0, -1}, {0, 1}},  // 1
        {{-1, 0}, {1, 0}},  // 2
        {{0, -1}, {1, 0}},  // 3
        {{0, 1}, {1, 0}},   // 4
        {{0, -1}, {-1, 0}}, // 5
        {{0, 1}, {-1, 0}}   // 6
    };

    public boolean hasValidPath(int[][] grid) {
        int m = grid.length, n = grid[0].length;
        boolean[][] visited = new boolean[m][n];
        
        return dfs(0, 0, m, n, grid, visited);
    }

    private boolean dfs(int r, int c, int m, int n, int[][] grid, boolean[][] visited) {
        // 到达右下角
        if (r == m - 1 && c == n - 1) {
            return true;
        }
        // 标记走过
        visited[r][c] = true;

        // 遍历当前街道支持的方向
        for (int[] dir : directions[grid[r][c]]) {
            int dr = dir[0], dc = dir[1];
            int nr = r + dr, nc = c + dc;

            // 边界检查
            if (nr >= 0 && nr < m && nc >= 0 && nc < n && !visited[nr][nc]) {
                // 连通性检查
                for (int[] backDir : directions[grid[nr][nc]]) {
                    if (backDir[0] == -dr && backDir[1] == -dc) {
                        if (dfs(nr, nc, m, n, grid, visited)) {
                            return true;
                        }
                    }
                }
            }
        }

        return false;
    }
}
```

- 时间复杂度： $O(n\times m)$，其中 $m,n$ 是数组 $grid$ 的行/列数，每个单元格最多访问一次
- 空间复杂度： $O(n\times m)$，最坏情况下递归栈的深度和 $visited$ 集合的大小均为网格大小
