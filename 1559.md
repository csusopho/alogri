[Problem: 1559. 二维网格图中探测环](https://leetcode.cn/problems/detect-cycles-in-2d-grid/description/)

### 方法：DFS & 并查集

将网格中的每个格子看作节点，如果相邻的两个格子值相同，则它们之间存在 **一条边**。那么，本题可以抽象为在无向图中寻找环。

图搜索，联想经典方法 $DFS$。

使用一个 $visited$ 集合记录已经访问过的节点。从任意一个 **未访问过** 的点出发进行 $DFS$。

在 $DFS$ 过程中，如果我们遇到了一个 **已经访问过** 的节点，且该节点 **不是当前节点的父节点**（即不是上一步走过来的那个点），那么就说明我们发现了一个环。

> 注意，这里的 “不是父节点” 是本题核心难点。不能简单判断 $visited$，不走回头路！

虽然题目要求长度 $\ge 4$ 且不能回退，但在网格中，如果能绕回到一个已访问的点（且非父节点），其路径长度自然会满足 $\ge 4$。

代码如下，已附加注释：

```Python
# python
class Solution:
    def containsCycle(self, grid: List[List[str]]) -> bool:
        m, n = len(grid), len(grid[0])
        visited = [[False] * n for _ in range(m)]

        # 四个方向：上，下，左，右
        directions = [(-1, 0), (1, 0), (0, -1), (0, 1)]

        def dfs(x, y, from_x, from_y, char):
            if visited[x][y]:
                return True  # 再次访问说明形成了环

            visited[x][y] = True
            for dx, dy in directions:
                nx, ny = x + dx, y + dy
                # 边界检查 + 不能回到上一个点 + 字符必须相同
                if 0 <= nx < m and 0 <= ny < n and (nx != from_x or ny != from_y) and grid[nx][ny] == char:
                    if dfs(nx, ny, x, y, char):
                        return True
            return False

        for i in range(m):
            for j in range(n):
                if not visited[i][j]:
                    if dfs(i, j, -1, -1, grid[i][j]):
                        return True
        return False
```

```Java
// java
class Solution {
    private int m, n;
    private boolean[][] visited;
    // 四个方向：上，下，左，右
    private int[][] directions = {{-1, 0}, {1, 0}, {0, -1}, {0, 1}};

    public boolean containsCycle(char[][] grid) {
        m = grid.length;
        n = grid[0].length;
        visited = new boolean[m][n];

        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                if (!visited[i][j]) {
                    if (dfs(i, j, -1, -1, grid[i][j], grid)) {
                        return true;
                    }
                }
            }
        }
        return false;
    }

    private boolean dfs(int x, int y, int from_x, int from_y, char target, char[][] grid) {
        if (visited[x][y]) {
            return true;  // 再次访问说明形成了环
        }

        visited[x][y] = true;
        for (int[] dir : directions) {
            int nx = x + dir[0];
            int ny = y + dir[1];
            // 边界检查 + 不能回到上一个点 + 字符必须相同
            if (nx >= 0 && nx < m && ny >= 0 && ny < n && (nx != from_x || ny != from_y) && grid[nx][ny] == target) {
                if (dfs(nx, ny, x, y, target, grid)) {
                    return true;
                }
            }
        }
        return false;
    }
}
```

- 时间复杂度： $O(n\times m)$，其中 $m,n$ 是数组 $grid$ 的行/列数，每个点和边访问常数次
- 空间复杂度： $O(n\times m)$，表示递归栈以及数组 $visited$ 的大小

### 并查集

换个角度看待本题，遍历图中所有的边。对于每条边 $(u, v)$，如果 $u$ 和 $v$ 已经属于同一个连通分量（即它们的根节点相同），那么加入这条边后就会 **形成环**。

> 也就是说，一个圆圈有一个缺口，补上这条边后，就形成环。

为了方便处理，将二维坐标 $(r, c)$ 映射为一维索引 $r * cols + c$。

遍历每个格子，只需要检查它的 **右侧** 和 **下侧** 节点。这样可以保证每条边只被处理一次！

- 只有当两个相邻格子值相同时，才尝试将它们合并。
- 如果在尝试合并前发现它们已经在同一个集合中，说明存在环。

并查集的具体原理/模版，可以看这篇题解，[传送门](https://leetcode.cn/problems/redundant-connection/solutions/2966966/bing-cha-ji-bao-li-mei-ju-zhao-huan-you-k08ct/)。

代码如下，已附加注释：

```Python
# python
class Solution:
    def containsCycle(self, grid: List[List[str]]) -> bool:
        n, m = len(grid), len(grid[0])
        parent = list(range(n * m))
        
        def find(i):
            if parent[i] == i:
                return i
            # 路径压缩
            parent[i] = find(parent[i])
            return parent[i]
        
        def union(i, j):
            root_i = find(i)
            root_j = find(j)
            if root_i != root_j:
                parent[root_i] = root_j
                return True
            # 已经在同一个集合中，说明是环
            return False
        
        for r in range(n):
            for c in range(m):
                # 检查右侧
                if c + 1 < m and grid[r][c] == grid[r][c+1]:
                    u = r * m + c
                    v = r * m + (c + 1)
                    if not union(u, v):
                        return True
                
                # 检查下方
                if r + 1 < n and grid[r][c] == grid[r+1][c]:
                    u = r * m + c
                    v = (r + 1) * m + c
                    if not union(u, v):
                        return True
         
        return False
```

```Java
// java
class Solution {
    private int[] parent;

    private int find(int i) {
        if (parent[i] == i) {
            return i;
        }
        // 路径压缩
        parent[i] = find(parent[i]);
        return parent[i];
    }

    private boolean union(int i, int j) {
        int root_i = find(i);
        int root_j = find(j);
        if (root_i != root_j) {
            parent[root_i] = root_j;
            return true;
        }
        // 已经在同一个集合中，说明是环
        return false;
    }

    public boolean containsCycle(char[][] grid) {
        int n = grid.length;
        int m = grid[0].length;
        parent = new int[n * m];
        for (int i = 0; i < n * m; i++) {
            parent[i] = i;
        }

        for (int r = 0; r < n; r++) {
            for (int c = 0; c < m; c++) {
                // 检查右侧
                if (c + 1 < m && grid[r][c] == grid[r][c + 1]) {
                    int u = r * m + c;
                    int v = r * m + (c + 1);
                    if (!union(u, v)) {
                        return true;
                    }
                }

                // 检查下方
                if (r + 1 < n && grid[r][c] == grid[r + 1][c]) {
                    int u = r * m + c;
                    int v = (r + 1) * m + c;
                    if (!union(u, v)) {
                        return true;
                    }
                }
            }
        }

        return false;
    }
}
```

- 时间复杂度： $O(m\times n)$，使用了路径压缩，单次操作的平均时间复杂度接近 $O(α(m\times n))$，其中 $α$ 是反阿克曼函数，其增长极其缓慢，对于所有实际问题都可以看作是一个极小的常数
- 空间复杂度： $O(m\times n)$，表示数组 $parent$ 的大小
