[Problem: 3225. 网格图操作后的最大分数](https://leetcode.cn/problems/maximum-score-from-grid-operations/description/)

### 方法：暴力 & DP优化

一个白色格子 $(i, j)$ 会得分，当且仅当它的左邻居 $(i, j-1)$ 或右邻居 $(i, j+1)$ 至少有一个是黑色的。

对第 $j$ 列选择下标 $(i, j)$，意味着将该列上方的前 $i+1$ 个格子变黑。用 $h_j$ 表示第 $j$ 列黑色的高度，即前 $h_j$ 行变黑，取值范围是 $[0, n]$。

当 $i> h_j$ 时，格子 $(i, j)$ 是白色的。它得分的条件是 $i < h_{j-1}$（左邻居为黑）或 $i < h_{j+1}$（右邻居为黑）。此时，第 $j$ 列的总得分为：

$$
\text{Score}_j = \sum_{i=h_j}^{\max(h_{j-1}, h_{j+1}) - 1} \text{grid}[i][j]
$$

若 $h_j \ge \max(h_{j-1}, h_{j+1})$，则得分为 $0$。

由于第 $j$ 列的得分取决于 $h_{j-1}$ 和 $h_{j+1}$，当我们处理到第 $j$ 列时，必须考虑左右两侧的高度。因此，**定义** $dp[j][curr\_h][prev\_h]$ 表示前 $j$ 列已经处理完，且第 $j$ 列高度为 $curr\_h$，第 $j-1$ 列高度为 $prev\_h$ 时的最大总分。

在计算第 $j$ 列时，我们实际上是为 **第 $j-1$ 列** 结算分数。因为只有知道了 $h_j$，第 $j-1$ 列的得分条件（由右邻居贡献的部分）才完整。

根据 $DP$ 的状态顺序，枚举前一列、当前列、后一列高度，得出 **转移方程**：

$$
dp[j+1][next\_h][curr\_h] = \max_{prev\_h} \{ dp[j][curr\_h][prev\_h] + \text{Score}(j, prev\_h, curr\_h, next\_h) \}
$$

其中 $\text{Score}(j, prev\_h, curr\_h, next\_h)$ 是第 $j$ 列在给定前后高度时的得分。

说白了，这种方法还是 **暴力**。枚举三列的高度，然后结算中间一列的数值。这里使用 $DP$ 形式，是为了方便后续解法的导出。

累加 $gird[起...终][j]$ 这一段的得分，**子数组** 求和问题，使用前缀和优化！具体原理，可以看这篇 [题解](https://leetcode.cn/problems/minimum-operations-to-make-all-array-elements-equal/solutions/2842529/pai-xu-qian-zhui-he-er-fen-fei-chang-qin-biux/)。这里定义 $presum[j][h]$ 表示第 $j$ 列上方 $h$ 个格子的前缀和。

代码如下，已附加注释：

```Python
# python
class Solution:
    def maximumScore(self, grid: List[List[int]]) -> int:
        n = len(grid)
        # presum[j][h] 表示第 j 列上方 h 个格子的前缀和
        presum = [[0] * (n + 1) for _ in range(n)]
        for j in range(n):
            for i in range(n):
                presum[j][i + 1] = presum[j][i] + grid[i][j]
        
        # dp[j][h][ph] 表示第 j 列高度为 h，第 j-1 列高度为 ph
        dp = [[[-float('inf')] * (n + 1) for _ in range(n + 1)] for _ in range(n + 1)]
        
        # 第 -1 列高度视为 0
        for h in range(n + 1):
            dp[0][h][0] = 0
        
        # 枚举当前列
        for j in range(1, n + 1):
            for curr_h in range(n + 1):  # 当前列 j 的高度
                for prev_h in range(n + 1):  # 第 j-1 列的高度
                    for pprev_h in range(n + 1):  # 第 j-2 列的高度
                        if dp[j-1][prev_h][pprev_h] == -float('inf'):
                            continue
                        
                        # 结算第 j-1 列的得分
                        score = 0
                        if j <= n:
                            # 第 j-1 列被 (j-2) 或 j 覆盖到的最高行
                            covered_h = max(pprev_h, curr_h if j < n else 0)
                            if covered_h > prev_h:
                                # 得分：从 prev_h 到 covered_h
                                score = presum[j-1][covered_h] - presum[j-1][prev_h]
                        # 更新
                        dp[j][curr_h][prev_h] = max(dp[j][curr_h][prev_h], 
                                                dp[j-1][prev_h][pprev_h] + score)
        
        # 最后一列（哨兵列）
        return max(max(row) for row in dp[n])
```

四重 $for$ 循环，时间复杂度为 $O(n^4)$。遗憾超时，卡在 $712/794$ 样例。

---

#### 转换思路

要降低复杂度，必须消除状态中 $prev\_h$ 和 $next\_h$ 的耦合。观察得分公式：

$$
\max(h_{j-1}, h_{j+1}) - h_j
$$

这个 $\max$ 是优化的阻碍。我们可以把网格的状态看作是一系列 “山峰” 和 “山谷”：

1. **递增段（爬坡）**： $h_{j-1} \le h_j$。此时第 $j-1$ 列会因为右边更高而得分。
2. **递减段（下坡）**： $h_{j-1} > h_j$。此时第 $j$ 列会因为左边更高而得分。

当前列的黑条长度 = 高度。递增，代表当前列 $j$ 的黑条高度 $>$ 前一列 $j-1$ 的黑条高度。

那么，将 $DP$ 状态拆分为两个趋势，从而不需要同时记录 $prev\_h$ 和 $next\_h$。

- $dp[j][h][0]$ (递增态)：表示第 $j$ 列高度为 $h$，且 $h \ge h_{j-1}$。
- $dp[j][h][1]$ (递减态)：表示第 $j$ 列高度为 $h$，且 $h < h_{j-1}$。

**场景 A：处理递增态 $dp[j][h][0]$**

如果当前是递增的（$h_j = h, h_{j-1} = k, k \le h$），那么第 $j-1$ 列的得分会增加。增加的部分是第 $j-1$ 列中从 $k$ 到 $h$ 的那段和。因此

$$
dp[j][h][0] = \max_{k \le h} \{ dp[j-1][k][0] + (\text{presum}[j-1][h] - \text{presum}[j-1][k]) \}
$$

这里只需要知道 $h \ge k$，不需要知道下一列 $h_{j+1}$。

**场景 B：处理递减态 $dp[j][h][1]$**

如果当前是递减的（$h_j = h, h_{j-1} = k, k > h$），那么第 $j$ 列的得分会增加。增加的部分是第 $j$ 列中从 $h$ 到 $k$ 的那段和。因此

$$
dp[j][h][1] = \max_{k > h} \{ \text{prev\_state} + (\text{presum}[j][k] - \text{presum}[j][h]) \}
$$

这里的 $prev\_state$ 可以是前一列的递增态或递减态。

也就是说，经过拆分逻辑，将三者之间的关系，变为相邻两者的关系！复杂度降至 $O(n^3)$。

---

#### 发现问题

注意，上述转移过程有 **严重逻辑错误**！

假设一共有三列，高度分别是 $h_{j-2}=4, h_{j-1}=0, h_j=4$。根据题目，第 $j-1$ 列（中间那一列）的得分是：

$$
\text{Sum}(\text{grid}[h_{j-1} : \max(h_{j-2}, h_j)][j-1])
$$

即 $\max(4, 4) - 0 = 4$ 行的分数。

但是，如果按照上面相邻的逻辑，简单地认为：

- 从 $j-2$ 到 $j-1$（下降）：第 $j-1$ 列得 $4-0=4$ 分。
- 从 $j-1$ 到 $j$（上升）：第 $j-1$ 列得 $4-0=4$ 分。

那么，第 $j-1$ 列被算了两次，总共得了 $8$ 行的分数。明显出错！

---

#### 解决方案

现在的问题：如何解决 “山谷” —— 下降+上升，倒 $V$ 形，重复累加的情况？

实际上，根据题意可知，真实逻辑为：当一列处于 “山谷” 底部时，它的分数由左边或右边 **“更高” 的那一列决定**，而不是相加。

因此，重新定义 $DP$，变为三个状态：

- **状态 0**：当前在 “爬坡” 或者在 “平地”。
- **状态 1**：当前在 “下坡”。
- **状态 2**：特殊的过渡态。表示刚刚结束了 “下坡”，现在准备开始 “爬坡”。

为了代码的美观，使用 **滚动数组** 的形式，将 $new\_dp$ 代表新的 $dp[j]$ 状态，而原有 $dp$ 代表旧的 $dp[j-1]$ 状态。

提前说明，前一列高度为 $k$，当前列高度为 $h$。

---

#### 状态转移

**1）** $new\_dp[h][0]$：爬坡态，当前高度 $h$ 比前一高度 $k$ 高（相等）。

当前列 $j$ 变高，第 $j-1$ 列中那些原本是白色的格子（高度在 $k$ 到 $h$ 之间的部分），现在因为右边（第 $j$ 列）变黑了，所以它们可以得分。此时有

$$
new\_dp[h][0] = max(dp[k][0], dp[k][2]) + (presum[j-1][h] - presum[j-1][k])
$$

可以从 “一直在爬坡” 转移来，也可以从 “谷底刚准备爬坡” 转移来。

**2）** $new\_dp[h][1]$：下坡态，当前高度 $h$ 比前一高度 $k$ 矮。

左边（第 $j-1$ 列）比当前列高，所以当前列 $j$ 中那些白色的格子（高度在 $h$ 到 $k$ 之间的部分），所以可以得分。此时有

$$
new\_dp[h][1] = max(dp[k][0], dp[k][1], dp[k][2]) + (presum[j][k] - presum[j][h])
$$

可以从任何状态转入下坡（从坡顶转下坡，或者继续下坡）。

**3）** $new\_dp[h][2]$：谷底态，专门处理 **“下坡完立即想爬坡”** 的情况。

强制让这个 “转折点” 不加分，得分为 $0$，解决重复计算！告诉 $DP$：“第 $j-1$ 列的分数之前已经领过了，现在开始爬坡时，先不要重复领”。此时有

$$
new\_dp[h][2] = max(dp[k][1])
$$

只能从前一列的 “下坡态” $dp[k][1]$ 转移来。

只要前一列是处于下坡趋势，无论它具体高度 $k$ 是多少，我们都取那个状态下最大的分值，平移到当前列的 $state2$。这相当于在谷底做了一个 “无损切换”。那么，数值 $k\in [0,n]$。

不好理解？

在状态 $2$ 里的高度 $h$，可以视为一个 “重新出发” 的起点。如果 $j+1$ 列比 $h$ 更高，它会按状态 $0$ 的逻辑开始加分，但由于它从状态 $2$ 转移，它避开了与 $h_{j-2}$ 那个高峰的重叠计算。

---

#### 总结

如果一直增： $state 0$ 会不断结算 **右邻居** 贡献的分。

如果一直减： $state 1$ 会不断结算 **左邻居** 贡献的分。

如果先减后增：**减** 的时候， $state 1$ 结算了分。**转弯时**，通过 $state 2$ 切换，不加分，避开了重复领分（直接平移 = 取 $max$）。转弯后如果继续增， $state 0$ 会从 $state 2$ 接手。

代码如下，已附加注释：

```Python
# python
class Solution:
    def maximumScore(self, grid: List[List[int]]) -> int:
        n = len(grid)
        # 列前缀和
        presum = [[0] * (n + 1) for _ in range(n)]
        for j in range(n):
            for i in range(n):
                presum[j][i + 1] = presum[j][i] + grid[i][j]
        
        # dp[h][0/1/2] 表示第 j 列高度为 h，且处于递增/递减/山谷趋势
        dp = [[-float('inf')] * 3 for _ in range(n + 1)]
        
        # 初始化第 0 列
        for h in range(n + 1):
            dp[h][0] = 0
        
        # 枚举第 j 列的高度 h
        for j in range(1, n):
            new_dp = [[-float('inf')] * 3 for _ in range(n + 1)]
            for h in range(n + 1):
                # 当前列更高，递增
                for k in range(h + 1):
                    prev_val = max(dp[k][0], dp[k][2])
                    if prev_val != -float('inf'):
                        score = presum[j-1][h] - presum[j-1][k]
                        new_dp[h][0] = max(new_dp[h][0], prev_val + score)
                
                # 当前列更低，递减
                for k in range(h + 1, n + 1):
                    prev_val = max(dp[k][0], dp[k][1], dp[k][2])
                    if prev_val != -float('inf'):
                        score = presum[j][k] - presum[j][h]
                        new_dp[h][1] = max(new_dp[h][1], prev_val + score)
                
                # 为防止重复计算，强制从 state 1 转移，不加分
                for k in range(n + 1):
                    if dp[k][1] != -float('inf'):
                        new_dp[h][2] = max(new_dp[h][2], dp[k][1])
            # 滚动更新
            dp = new_dp
        
        # 最终结果
        ans = 0
        for h in range(n + 1):
            ans = max(ans, dp[h][0], dp[h][1], dp[h][2])
        return ans
```

```Java
// java
class Solution {
    public long maximumScore(int[][] grid) {
        int n = grid.length;
        // 列前缀和
        long[][] presum = new long[n][n + 1];
        for (int j = 0; j < n; j++) {
            for (int i = 0; i < n; i++) {
                presum[j][i + 1] = presum[j][i] + grid[i][j];
            }
        }

        long NEG_INF = -1_000_000_000_000_000L;
        // dp[h][0/1/2] 表示第 j 列高度为 h，且处于递增/递减/山谷趋势
        long[][] dp = new long[n + 1][3];
        for (int h = 0; h <= n; h++) {
            Arrays.fill(dp[h], NEG_INF);
            // 初始化第 0 列
            dp[h][0] = 0;
        }

        // 枚举第 j 列的高度 h
        for (int j = 1; j < n; j++) {
            long[][] newDp = new long[n + 1][3];
            for (int h = 0; h <= n; h++) {
                Arrays.fill(newDp[h], NEG_INF);
            }

            for (int h = 0; h <= n; h++) {
                // 当前列更高，递增
                for (int k = 0; k <= h; k++) {
                    long prevVal = Math.max(dp[k][0], dp[k][2]);
                    if (prevVal > NEG_INF) {
                        long score = presum[j - 1][h] - presum[j - 1][k];
                        newDp[h][0] = Math.max(newDp[h][0], prevVal + score);
                    }
                }

                // 当前列更低，递减
                for (int k = h + 1; k <= n; k++) {
                    long prevVal = Math.max(dp[k][0], Math.max(dp[k][1], dp[k][2]));
                    if (prevVal > NEG_INF) {
                        long score = presum[j][k] - presum[j][h];
                        newDp[h][1] = Math.max(newDp[h][1], prevVal + score);
                    }
                }

                // 为防止重复计算，强制从 state 1 转移，不加分
                for (int k = 0; k <= n; k++) {
                    if (dp[k][1] > NEG_INF) {
                        newDp[h][2] = Math.max(newDp[h][2], dp[k][1]);
                    }
                }
            }
            // 滚动更新
            dp = newDp;
        }

        // 最终结果
        long ans = 0;
        for (int h = 0; h <= n; h++) {
            ans = Math.max(ans, Math.max(dp[h][0], Math.max(dp[h][1], dp[h][2])));
        }
        return ans;
    }
}
```

- 时间复杂度： $O(n^3)$，其中 $n$ 是正方形 $grid$ 的边长
- 空间复杂度： $O(n^2)$，表示数组 $presum$ 的大小
