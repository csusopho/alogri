[Problem: 2833. 距离原点最远的点](https://leetcode.cn/problems/furthest-point-from-origin/description/)

### 方法：贪心

注意，题目要求的是 $n$ 次移动后的 **最终** 位置，不是中间每一步能达到的位置。

因此，想到 **贪心** 策略：为了让最后的距离 $|pos|$ 最大，有两种极端策略：

1. **尽量向右走**：所有的 $\_$ 都当做 $R$。
2. **尽量向左走**：所有的 $\_$ 都当做 $L$。

设字符串中 $L,R,\_$ 的数量为 $count\_L,count\_R,count\_U$。

- 向右最远，最终坐标 $Pos_{right} = (count\_R - count\_L) + count\_U$
- 向左最远，最终坐标 $Pos_{left} = (count\_R - count\_L) - count\_U$

那么，最终结果是： $\max(|Pos_{right}|, |Pos_{left}|)$。

观察上面的公式，可以发现，无论 $count\_R - count\_L$ 是正数还是负数，为了让绝对值最大，我们只需要把所有的 $\_$ 都加到绝对值最大的方向上。

也即 **最大距离** = $|count\_R - count\_L| + count\_U$。

代码如下，已附加注释：

```Python
# python
class Solution:
    def furthestDistanceFromOrigin(self, moves: str) -> int:
        # 统计数量
        count_L = moves.count('L')
        count_R = moves.count('R')
        count_U = moves.count('_')
        
        # 净位移，再加上额外位移
        return abs(count_L - count_R) + count_U
```

```Java
// java
class Solution {
    public int furthestDistanceFromOrigin(String moves) {
        int count_L = 0;
        int count_R = 0;
        int count_U = 0;

        // 统计数量
        for (char c : moves.toCharArray()) {
            if (c == 'L') {
                count_L++;
            } else if (c == 'R') {
                count_R++;
            } else if (c == '_') {
                count_U++;
            }
        }

        // 净位移，再加上额外位移
        return Math.abs(count_L - count_R) + count_U;
    }
}
```

- 时间复杂度： $O(n)$，其中 $n$ 是串 $moves$ 的长度，一次遍历
- 空间复杂度： $O(1)$，仅使用常数个额外变量
