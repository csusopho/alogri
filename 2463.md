[Problem: 2463. 最小移动总距离](https://leetcode.cn/problems/minimum-total-distance-traveled/description/)

### 方法：DP

#### 观察

假设有两个机器人 $R_1, R_2$（位置 $x_1 < x_2$）和两个工厂位置 $F_1, F_2$（位置 $y_1 < y_2$）。

- 如果 $R_1 \to F_2$ 且 $R_2 \to F_1$，路径会发生交叉。
- 如果交换一下，让 $R_1 \to F_1$ 且 $R_2 \to F_2$，总距离一定会 **小于或等于** 交叉的情况（三角不等式在数轴上的体现）。

如果我们将机器人按位置排序，工厂也按位置排序，那么 **连续的一段机器人必然会分配给连续的一段工厂容量**。即机器人的相对顺序在维修后依然保持不变 —— 不交叉！

为什么不选贪心？

贪心是指 “每个机器人选最近的工厂”。但如果一个工厂容量有限，你占了某个位置，可能会迫使另一个机器人跑极远的距离。由于局部最优会导致全局糟糕，贪心失效。

怎么想到动规？

如果机器人和工厂都排好序，那么分配方案一定是 “连续的”。这意味着：如果我们决定了第 $i$ 个工厂修了 $k$ 个机器人，那么这 $k$ 个机器人一定是当前剩下的机器人里 **最右边** 的那几个。

这种 “一个接着一个” 做决策、且后一步决策依赖前一步结果的特征，是 $DP$ 的典型标志。

---

#### $DP$ 定义

要解决两个集合的匹配：机器人集合（大小 $n$）和工厂集合（大小 $m$）。

为了描述这个问题的状态，你至少得知道：现在处理到哪个工厂了？已经有多少个机器人被修好了？因此，维度自然就是 $(i, j)$。

$DP$ 的核心是：当我考虑第 $i$ 个工厂时，不需要知道前 $i-1$ 个工厂具体是怎么修机器人的。

- 只需要知道：前 $i-1$ 个工厂 **总共** 修了多少个机器人（假设是 $j-k$ 个）。
- 至于那 $j-k$ 个机器人是怎么分配给那 $i-1$ 个工厂的，我不在乎，我只要那个 “最小距离和”。

因此，**定义** $dp[i][j]$ 表示前 $i$ 个工厂修理了前 $j$ 个机器人所花费的最小总距离。

这种 “前缀状态” 的定义（前 $i$ 前 $j$）能完美剥离过去与未来，符合 $DP$ 的 **无后效性**。

---

#### $DP$ 转移

想象你是厂长，你面前有排好序的 $m$ 个工厂和 $n$ 个机器人。你正在考虑 **最后一个工厂**（第 $i$ 个）：

-）**选项 0：** 不让这个工厂修机器人。

那么任务就变成了：用前 $i-1$ 个工厂去修好全部 $j$ 个机器人。代价就是 $dp[i-1][j]$。

-）**选项 1：** 让这个工厂修 $1$ 个机器人。

根据排序原则，它修的必须是第 $j$ 个机器人。代价就是：前 $i-1$ 个工厂修前 $j-1$ 个机器人的代价 + 这个机器人到工厂的距离。也即 $dp[i-1][j-1] + \text{dist}(robot[j])$。

-）**选项 k：** 让这个工厂修 $k$ 个机器人（ $k \le limit$）。

它必须修最后面的第 $j-k+1$ 到第 $j$ 个机器人。代价就是：前 $i-1$ 个工厂修前 $j-k$ 个机器人的代价 + 这 $k$ 个机器人到该工厂的距离。也即 $dp[i-1][j-k] + \sum \text{dist}(j-k, j)$。

目标是最小化，所以遍历所有可能的 $k$，取最小值。**转移方程：**

$$
dp[i][j] = \min_{0 \le k \le \min(j, limit)} \{ dp[i-1][j-k] + \text{dist}(j-k, j) \}
$$

其中 $\text{dist}$ 是指第 $j-k$ 到 $j-1$ 个机器人到当前工厂位置的总距离。

边界条件：

- $dp[0][0] = 0$： $0$ 个工厂修 $0$ 个机器人距离为 $0$。
- $dp[0][j] = +\infty$：没有工厂无法修理机器人。
- $dp[i][0] = 0$：任意数量工厂修理 $0$ 个机器人距离为 $0$。

代码如下，已附加注释：

```Python
# python
class Solution:
    def minimumTotalDistance(self, robot: List[int], factory: List[List[int]]) -> int:
        # 排序
        robot.sort()
        factory.sort()
        n = len(robot)
        m = len(factory)

        # dp[i][j] 表示前 i 个工厂修理了前 j 个机器人所花费的最小总距离
        inf = float('inf')
        dp = [[inf] * (n + 1) for _ in range(m + 1)]
        
        # 边界
        for i in range(m + 1):
            dp[i][0] = 0
        
        # 递推
        for i in range(1, m + 1):
            pos, limit = factory[i-1]
            for j in range(1, n + 1):
                # 第 i 个工厂一个机器人也不修
                dp[i][j] = dp[i-1][j]
                # 第 i 个工厂修 k 个机器人
                cur_dist = 0
                
                # 受到当前总机器人数量 j 的限制
                for k in range(1, min(j, limit) + 1):
                    # 累加第 j-k 个机器人到当前工厂的距离
                    cur_dist += abs(robot[j-k] - pos)
                    # 前 i-1 个工厂修前 j-k 个机器人的最小距离 + 当前工厂修 k 个的距离
                    if dp[i-1][j-k] != inf:
                        dp[i][j] = min(dp[i][j], dp[i-1][j-k] + cur_dist)
        
        return dp[m][n]
```

```Java
// java
class Solution {
    public long minimumTotalDistance(List<Integer> robot, int[][] factory) {
        // 排序
        Collections.sort(robot);
        Arrays.sort(factory, (a, b) -> Integer.compare(a[0], b[0]));
        int n = robot.size();
        int m = factory.length;
        
        // dp[i][j] 表示前 i 个工厂修理了前 j 个机器人所花费的最小总距离
        long inf = Long.MAX_VALUE / 2;
        long[][] dp = new long[m + 1][n + 1];
        for (int i = 0; i <= m; i++) {
            Arrays.fill(dp[i], inf);
        }
        
        // 边界
        for (int i = 0; i <= m; i++) {
            dp[i][0] = 0;
        }
        
        // 递推
        for (int i = 1; i <= m; i++) {
            int pos = factory[i - 1][0];
            int limit = factory[i - 1][1];
            for (int j = 1; j <= n; j++) {
                // 第 i 个工厂一个机器人也不修
                dp[i][j] = dp[i - 1][j];
                // 第 i 个工厂修 k 个机器人
                long cur_dist = 0;
                
                // 受到当前总机器人数量 j 的限制
                for (int k = 1; k <= Math.min(j, limit); k++) {
                    // 累加第 j-k 个机器人到当前工厂的距离
                    cur_dist += Math.abs((long)robot.get(j - k) - pos);
                    // 前 i-1 个工厂修前 j-k 个机器人的最小距离 + 当前工厂修 k 个的距离
                    if (dp[i - 1][j - k] != inf) {
                        dp[i][j] = Math.min(dp[i][j], dp[i - 1][j - k] + cur_dist);
                    }
                }
            }
        }
        
        return dp[m][n];
    }
}
```

- 时间复杂度： $O(m\times n^2)$，其中 $n,m$ 是列表 $robot$、数组 $factory$ 的长度
- 空间复杂度： $O(m\times n)$，表示数组 $dp$ 的大小

这种 “序列前缀匹配” 的 $DP$ 定义方式在很多题目中都很常见，比如：

- 编辑距离： $dp[i][j]$ 表示字符串 $A$ 前 $i$ 个字符变成字符串 $B$ 前 $j$ 个字符的最小步数。
- 背包问题： $dp[i][j]$ 表示前 $i$ 件物品放入容量为 $j$ 的背包的最大价值。

本题的 $DP$ 本质相同：它是一种 “分配/匹配类” 问题的通用建模方式。
