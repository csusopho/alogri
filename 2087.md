[Problem: 2087. 网格图中机器人回家的最小代价](https://leetcode.cn/problems/minimum-cost-homecoming-of-a-robot-in-a-grid/description/)

### 方法：贪心

我们先来看看题目中的规则：

- 移动到第 $r$ 行，代价是 $rowCosts[r]$。
- 移动到第 $c$ 列，代价是 $colCosts[c]$。

这意味着：进入某一行或某一列的代价只取决于那一行的索引，而与你 **从哪个格子进入、或者在进入前走了多远完全无关。**

假设机器人要从第 $2$ 行移动到第 $5$ 行。

- **路径 A**：$2 \to 3 \to 4 \to 5$。总行代价 = $rowCosts[3] + rowCosts[4] + rowCosts[5]$。
- **路径 B**：$2 \to 1 \to 2 \to 3 \to 4 \to 5$（走了一段弯路）。总行代价 = $rowCosts[1] + rowCosts[2] + rowCosts[3] + rowCosts[4] + rowCosts[5]$。

题目限制代价是 **非负数**，所以任何 “回头路” 或者 “绕远路” 都会增加额外的代价。为了使代价最小，机器人应该 **只朝着目标方向直线移动**。

---

**为什么不能用 $DP$？**

通常在网格中使用 $DP$ 是因为：

1. 只能 **向右或向下**：这种限制保证了无后效性。
2. 每个格子的代价 **固定**：进入格子 $(r, c)$ 的代价是一个固定值。

但在本题中：

1. 方向 **不限**：机器人可以上下左右移动。如果有负权边，甚至会产生无限环路（虽然本题代价非负）。
2. 行/列代价 **独立**：垂直移动的代价只和行索引有关，水平移动的代价只和列索引有关。

也就是说，如果我们尝试建立一个 $dp[i][j]$ 表示到达 $(i, j)$ 的最小代价。那么

$$
dp[i][j] = \min(dp[i-1][j] + rowCosts[i], dp[i+1][j] + rowCosts[i], \dots)
$$

这实际上变成了一个图论的最短路径问题 $Dijkstra$ 算法。复杂度是 $O(mn \log(mn))$，而本题中网格大小 $m,n$ 可达 $10^5$，那么 $DP$ 或 $Dijkstra$ 空间和时间都会爆炸。

**本质原因**：无论先走完所有行再走列，还是行列交替走（锯齿形），只要不走回头路，经过的行和列的集合是固定的。

在本题的代价模型下，**从 $start$ 到 $home$ 的任何 “单调” 路径**，即只往家的方向走不回头，总代价都是完全一样的。

---

**贪心做法**：既然必须经过 $startRow$ 到 $homeRow$ 之间的所有行，且经过的每一行代价固定，那只需要计算：

1. 从 $startPos[0]$ 到 $homePos[0]$ 之间所有行的代价和（不包含起点行）。
2. 从 $startPos[1]$ 到 $homePos[1]$ 之间所有列的代价和（不包含起点列）。

别陷入 “思维陷阱”，一看到网格图就使用 $DP$ 或 $Dijkstra$，多浏览一眼数据范围吧。

代码如下，已附加注释：

```Python
# python
class Solution:
    def minCost(self, startPos: List[int], homePos: List[int], rowCosts: List[int], colCosts: List[int]) -> int:
        ans = 0
        
        start_row, start_col = startPos
        home_row, home_col = homePos
        
        # 计算行的代价
        if start_row < home_row:
            # 向下走
            for r in range(start_row + 1, home_row + 1):
                ans += rowCosts[r]
        elif start_row > home_row:
            # 向上走
            for r in range(home_row, start_row):
                ans += rowCosts[r]
        
        # 计算列的代价
        if start_col < home_col:
            # 向右走
            for c in range(start_col + 1, home_col + 1):
                ans += colCosts[c]
        elif start_col > home_col:
            # 向左走
            for c in range(home_col, start_col):
                ans += colCosts[c]
        
        return ans
```

```Java
// java
class Solution {
    public int minCost(int[] startPos, int[] homePos, int[] rowCosts, int[] colCosts) {
        int ans = 0;
        
        int start_row = startPos[0];
        int start_col = startPos[1];
        int home_row = homePos[0];
        int home_col = homePos[1];
        
        // 计算行的代价
        if (start_row < home_row) {
            // 向下走
            for (int r = start_row + 1; r <= home_row; r++) {
                ans += rowCosts[r];
            }
        } else if (start_row > home_row) {
            // 向上走
            for (int r = home_row; r < start_row; r++) {
                ans += rowCosts[r];
            }
        }
        
        // 计算列的代价
        if (start_col < home_col) {
            // 向右走
            for (int c = start_col + 1; c <= home_col; c++) {
                ans += colCosts[c];
            }
        } else if (start_col > home_col) {
            // 向左走
            for (int c = home_col; c < start_col; c++) {
                ans += colCosts[c];
            }
        }
        
        return ans;
    }
}
```

- 时间复杂度： $O(m+n)$，其中 $n,m$ 分别是数组 $colCosts,rowCosts$ 长度，线性遍历
- 空间复杂度： $O(1)$，仅使用常数个额外变量
