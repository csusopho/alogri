[Problem: 3070. 元素和小于等于 k 的子矩阵的数目](https://leetcode.cn/problems/count-submatrices-with-top-left-element-and-sum-less-than-k/description/)

### 方法：二维 & 一维前缀和

所有子矩阵必须包含 **左上角** $(0,0)$，因此每一个子矩阵都可以唯一地由其 **右下角** $(i, j)$ 来确定。目标就是找到所有满足 $\sum_{r=0}^i \sum_{c=0}^j \text{grid}[r][c] \le k$ 的索引对 $(i, j)$。

如何快速计算任意矩形区域的元素和？二维前缀和原理/推导，可以看这篇 [图解](https://leetcode.cn/problems/range-sum-query-2d-immutable/solutions/2993541/tu-jie-er-wei-qian-zhui-he-tui-dao-ji-su-61oc/)。

将 $(0,0)$ 为左上角 $(i,j)$ 为右下角的子矩阵和记为 $S[i][j]$，它可以通过动态规划的思想计算：

根据 **容斥原理**，要计算 $S[i][j]$，可以利用已经算出的上方和左侧的结果：

$$
S[i][j] = S[i-1][j] + S[i][j-1] - S[i-1][j-1] + \text{grid}[i][j]
$$

其中 $S[i-1][j]$ 表示上方矩形的和， $S[i][j-1]$ 表示左侧矩形的和， $S[i-1][j-1]$ 表示左上方重叠部分的和（被加了两次，所以要减去一次）。

**总结**：只需要一次遍历，计算所有子矩阵的和，然后判断每个子矩阵是否满足和 $\leq k$。

代码如下，已附加注释：

```Python
# python
class Solution:
    def countSubmatrices(self, grid: List[List[int]], k: int) -> int:
        m, n = len(grid), len(grid[0])
        prefix = [[0] * n for _ in range(m)]
        ans = 0
        
        for i in range(m):
            for j in range(n):
                # 计算当前子矩阵 (0,0) 到 (i,j) 的和
                cur_sum = grid[i][j]
                if i > 0:
                    cur_sum += prefix[i-1][j]
                if j > 0:
                    cur_sum += prefix[i][j-1]
                if i > 0 and j > 0:
                    cur_sum -= prefix[i-1][j-1]
                
                prefix[i][j] = cur_sum
                
                # 满足
                if prefix[i][j] <= k:
                    ans += 1
        
        return ans
```

```Java
// java
class Solution {
    public int countSubmatrices(int[][] grid, int k) {
        int m = grid.length, n = grid[0].length;
        int[][] prefix = new int[m][n];
        int ans = 0;
        
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                // 计算当前子矩阵 (0,0) 到 (i,j) 的和
                int cur_sum = grid[i][j];
                if (i > 0) {
                    cur_sum += prefix[i - 1][j];
                }
                if (j > 0) {
                    cur_sum += prefix[i][j - 1];
                }
                if (i > 0 && j > 0) {
                    cur_sum -= prefix[i - 1][j - 1];
                }
                
                prefix[i][j] = cur_sum;
                
                // 满足
                if (prefix[i][j] <= k) {
                    ans += 1;
                }
            }
        }
        
        return ans;
    }
}
```

- 时间复杂度： $O(m\times n)$，其中 $m,n$ 是矩阵 $grid$ 的行数、列数
- 空间复杂度： $O(m\times n)$，表示数组 $prefix$ 的大小

### 优化

想一想 **拼积木** 的过程，一个大矩形，不就是多个小矩形拼接的结果？

因此，我们可以将 $a\times b$ 大小的矩阵，看作 $b$ 个高度为 $a$ 的竖形矩阵拼接而来。

替换上面二维的思路，维护一个长度为 $n$ 的一维数组 $colSum$：

- $colSum[j]$ 表示在当前行 $i$ 时，第 $j$ 列从第 $0$ 行到第 $i$ 行的 **纵向** 元素之和，即 $colSum[j] = sum(grid[0...i][j])$
- 处理到第 $i$ 行时：更新 $colSum[j] += grid[i][j]$。同时，计算 $colSum$ 的 **横向累计前缀和**。设 $rowSum$ 为当前行从第 $0$ 列到第 $j$ 列的 $colSum$ 累加值。这个 $rowSum$ 实际上就是子矩阵 $(0,0)$ 到 $(i,j)$ 的总和。

也就是说，二维的累加和，换成了多个一维累加和的总和。

代码如下，已附加注释：

```Python
# python
class Solution:
    def countSubmatrices(self, grid: List[List[int]], k: int) -> int:
        m, n = len(grid), len(grid[0])
        # col_sums[j] 存储第 j 列第 0 行到第 i 行的纵向累加和
        col_sums = [0] * n
        ans = 0
        
        for i in range(m):
            row_acc = 0  # 存储 (0,0) 到 (i,j) 的实时总和
            for j in range(n):
                # 更新纵向累加和
                col_sums[j] += grid[i][j]
                # 统计列和
                row_acc += col_sums[j]
                
                if row_acc <= k:
                    ans += 1
        
        return ans
```

```Java
// java
class Solution {
    public int countSubmatrices(int[][] grid, int k) {
        int m = grid.length, n = grid[0].length;
        // col_sums[j] 存储第 j 列第 0 行到第 i 行的纵向累加和
        int[] col_sums = new int[n];
        int ans = 0;
        
        for (int i = 0; i < m; i++) {
            int row_acc = 0;  // 存储 (0,0) 到 (i,j) 的实时总和
            for (int j = 0; j < n; j++) {
                // 更新纵向累加和
                col_sums[j] += grid[i][j];
                // 统计列和
                row_acc += col_sums[j];
                
                if (row_acc <= k) {
                    ans += 1;
                }
            }
        }
        
        return ans;
    }
}
```

- 时间复杂度： $O(m\times n)$
- 空间复杂度： $O(n)$
