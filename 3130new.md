[Problem: 3130. 找出所有稳定的二进制数组 II](https://leetcode.cn/problems/find-all-possible-stable-binary-arrays-i/description/)

### 方法：记忆化 & DP

前排提示，本题是 $2024.8.7$ 的每日一题，我写过题解，链接为 [这篇](https://leetcode.cn/problems/find-all-possible-stable-binary-arrays-ii/solutions/2870953/ling-shen-ti-jie-xi-jie-chan-shu-ji-yi-h-ukd6/)。现在重新回味这一题，有了更多的感想，这里换成一种新思路，两篇题解内容完全不一样！

最直观 **暴力** 的解法：逐位构建这个二进制数组。在每一步，既可以选择放入 $0$，也可以选择放入 $1$。

为了满足题意，我们需要记录以下状态：

1. $z$：剩余可以放的 $0$ 的个数。
2. $o$：剩余可以放的 $1$ 的个数。
3. $last$：上一个放入的数字 $0/1$。
4. $cnt$：当前连续相同数字的长度。

写出如下代码：

```Python
# python
class Solution:
    def numberOfStableArrays(self, zero: int, one: int, limit: int) -> int:
        MOD = 10**9 + 7
        
        def dfs(z, o, last_val, cnt):
            # 边界条件：所有 0 和 1 都用完
            if z == 0 and o == 0:
                return 1
            
            res = 0
            
            # 尝试放 0
            if z > 0:
                if last_val == 0:
                    if cnt < limit:  # 不能超过限制
                        res = (res + dfs(z - 1, o, 0, cnt + 1)) % MOD
                else:
                    res = (res + dfs(z - 1, o, 0, 1)) % MOD
             
            # 尝试放 1
            if o > 0:
                if last_val == 1:
                    if cnt < limit:  # 不能超过限制
                        res = (res + dfs(z, o - 1, 1, cnt + 1)) % MOD
                else:
                    res = (res + dfs(z, o - 1, 1, 1)) % MOD
            
            return res
        
        # 一开始什么都没放
        return dfs(zero, one, -1, 0)
```

复杂度近似于 $O(2^{zero + one})$，一定会超时。

---

逐位放置 $0$ 和 $1$ 导致状态数太多，我们可以转换视角：**不要一个一个数字放，而是一块一块地放。**

合法的数组一定是由一段连续的 $0$、一段连续的 $1$ 交替组成的，且每一段的长度在 $[1, limit]$ 之间。重新定义状态为 $dfs(z, o, lastType)$

- $z$：剩余的 $0$ 的数量。
- $o$：剩余的 $1$ 的数量。
- $lastType$：上一次放置的段是 $0/1$。

**状态转移**：

- 如果上一段是 $1$（或者一开始），接下来必须放一段 $0$，且放 $0$ 的数量 $k \in[1, \min(z, \text{limit})]$。
- 如果上一段是 $0$，接下来必须放一段 $1$，且放 $1$ 的数量 $k \in[1, \min(o, \text{limit})]$。

整个递归中有大量重复递归调用（递归参数相同）。由于 **递归函数没有副作用**，同样的入参无论计算多少次，算出来的结果都是一样的，因此可以用记忆化搜索来优化。

记忆化的 **本质** 就是减少分支数，避免重复搜索，降低时间复杂度。递归的参数只有两个，所以记忆化数组需要两维。 $Java$ 中使用数组记录， $Python$ 中直接使用 $@cache$ 装饰器即可。

修改上面的代码，整体的框架不变，已附加注释：

```Python
# python
class Solution:
    def numberOfStableArrays(self, zero: int, one: int, limit: int) -> int:
        MOD = 10**9 + 7
        
        # 剩余0数量，剩余1数量，上次是什么数字
        @cache
        def dfs(z, o, last_type):
            # 边界条件：所有 0 和 1 都用完
            if z == 0 and o == 0:
                return 1
            
            res = 0
            if last_type == 1:
                # 必须放 0，枚举放几个 0
                for k in range(1, min(z, limit) + 1):
                    res = (res + dfs(z - k, o, 0)) % MOD
            else:
                # 必须放 1，枚举放几个 1
                for k in range(1, min(o, limit) + 1):
                    res = (res + dfs(z, o - k, 1)) % MOD
                  
            return res
        
        # 第一段任意
        return (dfs(zero, one, 1) + dfs(zero, one, 0)) % MOD
```

```Java
// java
class Solution {
    int MOD = 1000000007;
    int limit;
    int[][][] memo;

    public int numberOfStableArrays(int zero, int one, int limit) {
        this.limit = limit;
        memo = new int[zero + 1][one + 1][2];
        for (int i = 0; i <= zero; i++) {
            for (int j = 0; j <= one; j++) {
                Arrays.fill(memo[i][j], -1);
            }
        }
        
        // 第一段任意
        return (dfs(zero, one, 1) + dfs(zero, one, 0)) % MOD;
    }

    // 剩余0数量，剩余1数量，上次是什么数字
    private int dfs(int z, int o, int last_type) {
        // 边界条件：所有 0 和 1 都用完
        if (z == 0 && o == 0) {
            return 1;
        }
        
        if (memo[z][o][last_type] != -1) {
            return memo[z][o][last_type];
        }

        int res = 0;
        if (last_type == 1) {
            // 必须放 0，枚举放几个 0
            for (int k = 1; k <= Math.min(z, limit); k++) {
                res = (res + dfs(z - k, o, 0)) % MOD;
            }
        } else {
            // 必须放 1，枚举放几个 1
            for (int k = 1; k <= Math.min(o, limit); k++) {
                res = (res + dfs(z, o - k, 1)) % MOD;
            }
        }

        return memo[z][o][last_type] = res;
    }
}
```

记忆化后，每个状态只会计算一次。时间复杂度 = 状态个数 $\times$ 单个状态的计算时间。参数一共三个，状态个数为 $zero \times one \times 2$，每个状态需要经过 $limit$ 次循环转移。

- 时间复杂度： $O(zero \times one \times limit)$
- 空间复杂度： $O(zero \times one)$，保存多少状态，就需要多少空间

注意，这段代码也会 **超时**！经实测，卡在 $566/582$ 样例。

---

### 差分 $DP$

上面 $O(limit)$ 的循环转移非常浪费时间，需要优化为 $O(1)$ 时间。

设 $dp[i][j][0]$ 为放置了 $i$ 个 $0$ 和 $j$ 个 $1$ 且 **以 0 结尾** 的方案数。设 $dp[i][j][1]$ 为放置了 $i$ 个 $0$ 和 $j$ 个 $1$ 且 **以 1 结尾** 的方案数。

根据第二步的分组思想， $dp[i][j][0]$ 等于在之前以 1 结尾的字符串后面，追加 $k$ 个 $0$：

$$
dp[i][j][0] = \sum_{k=1}^{\min(i, \text{limit})} dp[i-k][j][1]
$$

将 $i$ 替换为 $i-1$，可以得到：

$$
dp[i-1][j][0] = \sum_{k=1}^{\min(i-1, \text{limit})} dp[i-1-k][j][1]
$$

**错位相减法**：将一式减去二式，绝大部分中间项 $dp[i-2][j][1], dp[i-3][j][1] \dots$ 等等都会被完美抵消！移项整理后，得到：

$$
dp[i][j][0] = dp[i-1][j][0] + dp[i-1][j][1] - \begin{cases} dp[i - 1 - \text{limit}][j][1] & i > \text{limit} \\ 0 & i \le \text{limit} \end{cases}
$$

**直观通俗的理解：** 计算当前 $i$ 个 $0$，$j$ 个 $1$ 且以 $0$ 结尾的方案数，相当于我们在任意合法的 $(i-1, j)$ 序列尾部 **强行追加** 一个 $0$，即 $dp[i-1][j][0] + dp[i-1][j][1]$。

但强行追加一个 $0$ 可能会导致末尾出现了 $limit + 1$ 个连续的 $0$，这种非法情况必须被剔除。什么时候会出现恰好连续 $limit + 1$ 个 $0$？只有在追加前，末尾 **已经是恰好** 由 $1$ 结尾并紧接着 $limit$ 个 $0$ 时才会发生。这类非法前置状态的数量严格等于 $dp[i - 1 - limit][j][1]$！

**初始化**：当只有 $0$ 没有 $1$ 时，只要 $0$ 的数量不超过 $limit$，必然存在唯一合法序列。因此，将范围内的 $dp[i][0][0]$ 都设置为 $1$，同理 $dp[0][j][1]$。

代码如下，已附加注释：

```Python
# python
class Solution:
    def numberOfStableArrays(self, zero: int, one: int, limit: int) -> int:
        MOD = 10**9 + 7

        # dp[i][j][0]: 有 i 个 0，j 个 1，且最后一个字符是 0 的稳定数组数量
        # dp[i][j][1]: 有 i 个 0，j 个 1，且最后一个字符是 1 的稳定数组数量
        dp = [[[0, 0] for _ in range(one + 1)] for _ in range(zero + 1)]

        # 初始化
        for i in range(1, min(zero, limit) + 1):
            dp[i][0][0] = 1
        for j in range(1, min(one, limit) + 1):
            dp[0][j][1] = 1

        # 自底向上填表
        for i in range(1, zero + 1):
            for j in range(1, one + 1):
                # 追加一个 0
                dp[i][j][0] = (dp[i-1][j][0] + dp[i-1][j][1]) % MOD
                # 剔除超限的非法情况
                if i > limit:
                    dp[i][j][0] = (dp[i][j][0] - dp[i - limit - 1][j][1] + MOD) % MOD

                # 追加一个 1
                dp[i][j][1] = (dp[i][j-1][0] + dp[i][j-1][1]) % MOD
                # 剔除超限的非法情况
                if j > limit:
                    dp[i][j][1] = (dp[i][j][1] - dp[i][j - limit - 1][0] + MOD) % MOD

        # 汇总
        return (dp[zero][one][0] + dp[zero][one][1]) % MOD
```

```Java
// java
class Solution {
    public int numberOfStableArrays(int zero, int one, int limit) {
        int MOD = 1000000007;
        
        // dp[i][j][0]: 有 i 个 0，j 个 1，且最后一个字符是 0 的稳定数组数量
        // dp[i][j][1]: 有 i 个 0，j 个 1，且最后一个字符是 1 的稳定数组数量
        int[][][] dp = new int[zero + 1][one + 1][2];
        
        // 初始化
        for (int i = 1; i <= Math.min(zero, limit); i++) {
            dp[i][0][0] = 1;
        }
        for (int j = 1; j <= Math.min(one, limit); j++) {
            dp[0][j][1] = 1;
        }
        
        // 自底向上填表
        for (int i = 1; i <= zero; i++) {
            for (int j = 1; j <= one; j++) {
                // 追加一个 0
                dp[i][j][0] = (dp[i - 1][j][0] + dp[i - 1][j][1]) % MOD;
                // 剔除超限的非法情况
                if (i > limit) {
                    dp[i][j][0] = (dp[i][j][0] - dp[i - limit - 1][j][1] + MOD) % MOD;
                }
                
                // 追加一个 1
                dp[i][j][1] = (dp[i][j - 1][0] + dp[i][j - 1][1]) % MOD;
                // 剔除超限的非法情况
                if (j > limit) {
                    dp[i][j][1] = (dp[i][j][1] - dp[i][j - limit - 1][0] + MOD) % MOD;
                }
            }
        }
        
        // 汇总
        return (dp[zero][one][0] + dp[zero][one][1]) % MOD;
    }
}
```

- 时间复杂度： $O(zero\times one)$，两重 $for$ 循环
- 空间复杂度： $O(zero\times one)$，表示数组 $dp$ 的大小

### 空间优化

根据转移方程可知，在计算 $dp[i][j][0]$ 和 $dp[i][j][1]$ 时，第一维的行数 $i$，它最多只依赖于：

- 上一行： $i - 1$
- 之前的第 $limit + 1$ 行： $i - limit - 1$

那么，没必要开辟 $zero + 1$ 这么大的第一维度数组，只需要保留 **最近的** $limit + 1$ 行历史记录即可。利用取模运算 $i \% (limit + 1)$ 在固定大小的数组中循环覆盖。由于可能 $limit > zero$，因此 **窗口大小** 可以设定为 $R = min(zero, limit) + 1$。

如何更新状态？

计算到第 $i$ 行，准备写入 $idx = i \% R$ 的位置时，此时该位置存储的恰好就是 $limit + 1$ 行前，即第 $i - limit - 1$ 行的历史数据！因此，在覆盖当前格子前，我们只需把旧数据提取出来即可，省去了额外的存储。

**对称性压缩**： $0$ 和 $1$ 的逻辑是完全对称的。为了让内存进一步缩减，如果 $zero < one$，可以直接交换两个变量，使得矩阵的 “列数” 变得更小。

代码如下，已附加注释：

```Python
# python
class Solution:
    def numberOfStableArrays(self, zero: int, one: int, limit: int) -> int:
        MOD = 10**9 + 7

        # 对称性优化
        if zero < one:
            zero, one = one, zero

        # 滚动数组
        R = min(zero, limit) + 1
        dp = [[[0, 0] for _ in range(one + 1)] for _ in range(R)]
        
        # 初始化第 0 行
        for j in range(1, min(one, limit) + 1):
            dp[0][j][1] = 1

        # 自底向上填表
        for i in range(1, zero + 1):
            idx = i % R
            prev_idx = (i - 1) % R

            # 当前新行的第 0 列
            dp[idx][0][0] = 1 if i <= limit else 0
            dp[idx][0][1] = 0

            for j in range(1, one + 1):
                # 在覆盖 dp[idx][j][1] 之前，它的值恰好就是第 i - limit - 1 行留下来的历史记录
                old_1 = dp[idx][j][1]

                # 追加一个 0
                dp[idx][j][0] = (dp[prev_idx][j][0] + dp[prev_idx][j][1]) % MOD
                if i > limit:
                    dp[idx][j][0] = (dp[idx][j][0] - old_1 + MOD) % MOD

                # 追加一个 1
                dp[idx][j][1] = (dp[idx][j-1][0] + dp[idx][j-1][1]) % MOD
                if j > limit:
                    dp[idx][j][1] = (dp[idx][j][1] - dp[idx][j - limit - 1][0] + MOD) % MOD

        # 汇总
        return (dp[zero % R][one][0] + dp[zero % R][one][1]) % MOD
```

```Java
// java
class Solution {
    public int numberOfStableArrays(int zero, int one, int limit) {
        int MOD = 1000000007;

        // 对称性优化
        if (zero < one) {
            int tmp = zero;
            zero = one;
            one = tmp;
        }

        // 滚动数组
        int R = Math.min(zero, limit) + 1;
        int[][][] dp = new int[R][one + 1][2];

        // 初始化第 0 行
        for (int j = 1; j <= Math.min(one, limit); j++) {
            dp[0][j][1] = 1;
        }

        // 自底向上填表
        for (int i = 1; i <= zero; i++) {
            int idx = i % R;
            int prev_idx = (i - 1) % R;

            // 当前新行的第 0 列
            dp[idx][0][0] = (i <= limit) ? 1 : 0;
            dp[idx][0][1] = 0;

            for (int j = 1; j <= one; j++) {
                // 在覆盖 dp[idx][j][1] 之前，它的值恰好就是第 i - limit - 1 行留下来的历史记录
                int old_1 = dp[idx][j][1];

                // 追加一个 0
                dp[idx][j][0] = (dp[prev_idx][j][0] + dp[prev_idx][j][1]) % MOD;
                if (i > limit) {
                    dp[idx][j][0] = (dp[idx][j][0] - old_1 + MOD) % MOD;
                }

                // 追加一个 1
                dp[idx][j][1] = (dp[idx][j - 1][0] + dp[idx][j - 1][1]) % MOD;
                if (j > limit) {
                    dp[idx][j][1] = (dp[idx][j][1] - dp[idx][j - limit - 1][0] + MOD) % MOD;
                }
            }
        }

        // 汇总
        return (dp[zero % R][one][0] + dp[zero % R][one][1]) % MOD;
    }
}
```

- 时间复杂度： $O(zero\times one)$
- 空间复杂度： $O(min(zero,one,limit)\times min(zero,one))$
