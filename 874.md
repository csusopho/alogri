[Problem: 874. 模拟行走机器人](https://leetcode.cn/problems/walking-robot-simulation/description/)

### 方法：模拟 & 优化

1）如何表示方向？

一共四个面向：北、东、南、西。可以用坐标增量 $(dx, dy)$ 来表示：

- **北**： $(0, 1)$，也即 $y$ 轴正方向
- **东**： $(1, 0)$，也即 $x$ 轴正方向
- **南**： $(0, -1)$，也即 $y$ 轴负方向
- **西**： $(-1, 0)$，也即 $x$ 轴负方向

将这些方向按 **顺时针** 顺序放入一个列表，得到 $directions = [(0, 1), (1, 0), (0, -1), (-1, 0)]$。利用循环特性，这样处理转向就会非常简单：

- 向右转：索引在列表中向后移动一位 $(index + 1) \% 4$。
- 向左转：索引在列表中向前移动一位 $(index - 1) \% 4$。

2）如何快速判断障碍物？

如果机器人每走一步，都去遍历一遍列表$ obstacles$ 判断是否撞墙，效率非常低。

因此，将障碍物坐标转换成一个 $Set$ **集合**。如此一来，集合的查找效率是 $O(1)$，这样即使有上万个障碍物，判断速度也极快。

3）如何模拟移动？

对于每一个前进命令 $x$，机器人需要尝试走 $x$ 步。模拟 **每一步** 移动过程：

- 每一小步，先计算出下一个潜在坐标 $(nextX, nextY)$。
- 检查这个坐标是否在障碍物集合中。
- 如果 **不在**，更新当前位置，并更新最大距离的平方。
- 如果 **在**，立即停止当前的移动指令，留在原地。

代码如下，已附加注释：

```Python
# python
class Solution:
    def robotSim(self, commands: List[int], obstacles: List[List[int]]) -> int:
        # 北、东、南、西四个方向的增益
        dx = [0, 1, 0, -1]
        dy = [1, 0, -1, 0]
        
        x, y = 0, 0  # 起始点
        di = 0  # 初始方向索引，北方
        ans = 0  # 最大距离平方值
        # 将障碍物存入集合
        obstacle_set = {tuple(obs) for obs in obstacles}
        
        for cmd in commands:
            if cmd == -2:
                # 向左转
                di = (di - 1) % 4
            elif cmd == -1:
                # 向右转
                di = (di + 1) % 4
            else:
                # 尝试向前走 cmd 步
                for _ in range(cmd):
                    next_x = x + dx[di]
                    next_y = y + dy[di]
                    
                    # 检查下一个位置
                    if (next_x, next_y) not in obstacle_set:
                        # 没有障碍物，更新坐标
                        x, y = next_x, next_y
                        # 记录
                        ans = max(ans, x*x + y*y)
                    else:
                        # 遇到障碍物，停留在原地
                        break
        
        return ans
```

```Java
// java
class Solution {
    public int robotSim(int[] commands, int[][] obstacles) {
        // 北、东、南、西四个方向的增益
        int[] dx = {0, 1, 0, -1};
        int[] dy = {1, 0, -1, 0};
        
        int x = 0, y = 0;  // 起始点
        int di = 0;  // 初始方向索引，北方
        int ans = 0;  // 最大距离平方值
        // 将障碍物存入集合
        Set<Long> obstacleSet = new HashSet<>();
        for (int[] obs : obstacles) {
            obstacleSet.add(((long) obs[0] << 32) | (obs[1] & 0xFFFFFFFFL));
        }
        
        for (int cmd : commands) {
            if (cmd == -2) {
                // 向左转
                di = (di + 3) % 4;
            } else if (cmd == -1) {
                // 向右转
                di = (di + 1) % 4;
            } else {
                // 尝试向前走 cmd 步
                for (int i = 0; i < cmd; i++) {
                    int next_x = x + dx[di];
                    int next_y = y + dy[di];
                    
                    // 检查下一个位置
                    if (!obstacleSet.contains(((long) next_x << 32) | (next_y & 0xFFFFFFFFL))) {
                        // 没有障碍物，更新坐标
                        x = next_x;
                        y = next_y;
                        // 记录
                        ans = Math.max(ans, x * x + y * y);
                    } else {
                        // 遇到障碍物，停留在原地
                        break;
                    }
                }
            }
        }
        
        return ans;
    }
}
```

- 时间复杂度： $O(n+m)$，其中 $n,m$ 分别是数组 $commands,obstacles$ 的长度
- 空间复杂度： $O(m)$，表示哈希表 $set$ 的大小

### 优化

仔细对比两个数组 $dx$ 和 $dy$，可以发现： $dy$ 实际上就是 $dx$ 数组向左偏移一位的结果。

那么，只需要构造一个数组 $dirs = [0, 1, 0, -1, 0]$：

- **北**：有 $dx = dirs[0] = 0$, $dy = dirs[1] = 1$
- **东**：有 $dx = dirs[1] = 1$, $dy = dirs[2] = 0$
- **南**：有 $dx = dirs[2] = 0$, $dy = dirs[3] = -1$
- **西**：有 $dx = dirs[3] = -1$, $dy = dirs[4] = 0$

这种写法减少了变量维护，代码更简洁。

由于机器人在一个命令下做的是 **直线运动**，直线段上距离原点最远的点一定是在该段的起点或终点，因此只需在每条指令执行完后判断一次即可。

如果用几何数学来解释的话：

- 机器人每一条移动命令都是在做 **射线/线段运动**。
- 欧式距离的平方函数 $f(x, y) = x^2 + y^2$ 是一个凸函数。
- 在一条线段上，凸函数的最大值一定在该线段的 **两个端点** 取得。
- 因为当前的起点就是上一次移动的终点（上一次已经判断过了），所以我们只需要在本次移动结束（到达终点或撞到障碍物）时判断一次即可。

**结论**：这不仅减少了 $max()$ 函数的调用次数，且不会漏掉任何可能的最大值。

代码如下，已附加注释：

```Python
# python
class Solution:
    def robotSim(self, commands: List[int], obstacles: List[List[int]]) -> int:
        # 方向数组
        dirs = [0, 1, 0, -1, 0]
        
        x, y = 0, 0
        di = 0  # 起始索引
        ans = 0
        # 将障碍物存入集合
        obstacle_set = {tuple(obs) for obs in obstacles}
        
        for cmd in commands:
            if cmd == -2:  # 向左转
                di = (di - 1) % 4
            elif cmd == -1:  # 向右转
                di = (di + 1) % 4
            else:
                dx, dy = dirs[di], dirs[di+1]
                # 尝试移动 cmd 步
                for _ in range(cmd):
                    if (x + dx, y + dy) not in obstacle_set:
                        x += dx
                        y += dy
                    else:
                        # 遇到障碍物，跳出当前移动
                        break
                
                # 直线运动中，最远点只会在端点产生
                ans = max(ans, x * x + y * y)
        
        return ans
```

```Java
// java
class Solution {
    public int robotSim(int[] commands, int[][] obstacles) {
        // 方向数组
        int[] dirs = {0, 1, 0, -1, 0};
        
        int x = 0, y = 0;
        int di = 0;  // 起始索引
        int ans = 0;
        // 将障碍物存入集合
        Set<Long> obstacleSet = new HashSet<>();
        for (int[] obs : obstacles) {
            obstacleSet.add(((long) obs[0] << 32) | (obs[1] & 0xFFFFFFFFL));
        }
        
        for (int cmd : commands) {
            if (cmd == -2) {  // 向左转
                di = (di + 3) % 4;
            } else if (cmd == -1) {  // 向右转
                di = (di + 1) % 4;
            } else {
                int dx = dirs[di];
                int dy = dirs[di + 1];
                // 尝试移动 cmd 步
                for (int i = 0; i < cmd; i++) {
                    if (!obstacleSet.contains(((long) (x + dx) << 32) | ((y + dy) & 0xFFFFFFFFL))) {
                        x += dx;
                        y += dy;
                    } else {
                        // 遇到障碍物，跳出当前移动
                        break;
                    }
                }
                
                // 直线运动中，最远点只会在端点产生
                ans = Math.max(ans, x * x + y * y);
            }
        }
        
        return ans;
    }
}
```

- 空间复杂度： $O(m)$
