[Problem: 1415. 长度为 n 的开心字符串中字典序第 k 小的字符串](https://leetcode.cn/problems/the-k-th-lexicographical-string-of-all-happy-strings-of-length-n/description/)

### 方法：DFS & 数学

开心字符串的约束很简单：字符集是 ${a, b, c}$，且相邻字符不能相同。

先计算一下总数：

1. 第一个字符有 **三种** 选择 $a,b,c$。
2. 从第二个字符开始，为了不和前一个重复，每个位置都有 **两种** 选择。
3. 因此，长度为 $n$ 的开心字符串总共有 $3 \times 2^{n-1}$ 个。

我们可以把构造一个长度为 $n$ 的开心字符串看作一个 **$n$ 层的决策过程**：

- 根节点：空字符串 `""`。
- 第 $1$ 层：决定字符串的第 $1$ 个字符，可选 $a,b,c$。
- 第 2 层：决定字符串的第 $2$ 个字符，可选剩下的 $2$ 个不重复字符。
- $\dots$
- 第 $n$ 层：决定最后一个字符。

这就相当于一个决策树。从空字符串开始，**尽可能“深”地去构造一个完整的字符串**。只有当一条路走到底或者无路可走时，才 “回退” 到上一步，尝试另一个选项。

---

字典序的规则是：从左到右比较，**第一个** 不相等的字符决定了大小。

为了保证 $DFS$ 生成的顺序是字典序，我们必须满足两个关键条件：

1）顺序选择 $a,b,c$：

- 先尝试所有以 $a$ 开头的字符串。
- 全部尝试完后，才会去尝试以 $b$ 开头的字符串。
- 最后才尝试以 $c$ 开头的字符串。

2）先序遍历：

$DFS$ 遵循 “先深入，再并列” 的原则。对于字符串来说，这正好符合字典序的比较逻辑：**高位（前面的字符）的优先级高于低位（后面的字符）。**

以 $n=2$ 示范，此时 $DFS$ 的执行路径如下：

1. 第一位选 $a$
    - 第二位尝试 $a$ (跳过，因为相同)
    - 第二位尝试 $b$ -> 得到 $ab$ (第1个)
    - 第二位尝试 $c$ -> 得到 $ac$ (第2个)
2. 回溯到第一位，选 $b$
    - 第二位尝试 $a$ -> 得到 $ba$ (第3个)
    - 第二位尝试 $b$ (跳过)
    - 第二位尝试 $c$ -> 得到 $bc$ (第4个)
3. 回溯到第一位，选 $c$
    - 第二位尝试 $a$ -> 得到 $ca$ (第5个)
    - 第二位尝试 $b$ -> 得到 $cb$ (第6个)
    - 第二位尝试 $c$ (跳过)

会发现生成的顺序依次是： $ab,ac,ba,bc,ca,cb$。正好是这 6 个字符串的 **升序排列**。

---

可以把这个过程想象成走一个迷宫，迷宫的每个路口都有三扇门，分别贴着 $a,b,c$。

- 永远 **优先走左边** 的那扇门 $a$。
- 走到尽头时，记录下当前路径，这就是当前最小的字符串。
- 然后退回到最近的一个路口，走 **中间** 的那扇门 $b$。
- 接着再退回，走 **右边** 的那扇门 $c$。

因为始终保持 **“左 > 中 > 右”** 的探索顺序，所以找到完整路径的先后顺序，自然就是这些路径对应字符串的 **字典序从小到大**。

$DFS$ 在这里是：一种 **按位** 确定字符、逐条路径探索的搜索方式。我们在每一位尝试字符时，是按 $a -> b -> c$ 的顺序。在确定下一位之前，绝对不会跳到当前位的下一个选项（即先确定前缀，再确定后缀）。这两点结合，完美契合了字典序的定义。

代码如下，已附加注释：

```Python
# python
class Solution:
    def getHappyString(self, n: int, k: int) -> str:
        self.count = 0  # 当前串长
        self.ans = ""
        
        def backtrack(cur_str):
            # 已经找到
            if self.ans:
                return
            
            # 到达目标长度
            if len(cur_str) == n:
                self.count += 1
                if self.count == k:
                    self.ans = cur_str
                return
            
            # 顺序尝试
            for char in ['a', 'b', 'c']:
                # 拼接的字符与上一个字符不同
                if not cur_str or cur_str[-1] != char:
                    backtrack(cur_str + char)
        
        backtrack("")
        return self.ans
```

```Java
// java
class Solution {
    private int count = 0; // 当前串长
    private String ans = "";

    public String getHappyString(int n, int k) {
        backtrack("", n, k);
        return ans;
    }

    private void backtrack(String curStr, int n, int k) {
        // 已经找到
        if (!ans.equals("")) {
            return;
        }

        // 到达目标长度
        if (curStr.length() == n) {
            count++;
            if (count == k) {
                ans = curStr;
            }
            return;
        }

        // 顺序尝试
        for (char c : new char[]{'a', 'b', 'c'}) {
            // 拼接的字符与上一个字符不同
            if (curStr.length() == 0 || curStr.charAt(curStr.length() - 1) != c) {
                backtrack(curStr + c, n, k);
            }
        }
    }
}
```

- 时间复杂度： $O(k\times n)$，最坏情况需要生成 $k$ 个字符串，每个字符串长度为 $n$
- 空间复杂度： $O(n)$，表示递归栈的深度

### 数学

可以逐位确定，直接 “定位” 第 $k$ 个字符串，不需要生成前 $k-1$ 个字符串！

长度为 $n$ 的总数是 $3 \times 2^{n-1}$。如果 $k$ 大于这个数，直接返回空字符串。

**确定第 1 位**：可以选择 $a,b,c$。

- 以 $a$ 开头的字符串有 $2^{n-1}$ 个。
- 以 $b$ 开头的字符串有 $2^{n-1}$ 个。
- 以 $c$ 开头的字符串有 $2^{n-1}$ 个。

每种字母开头都是 $2^{n-1}$ 种可能，因此通过 `(k - 1) // 2^(n-1)` 确定第 $1$ 位是哪个字母。

**确定后续位 (第 $i$ 位)**：

- 一旦确定了前一位字符，当前位只有 **两个** 可选字符。
- 比如前一位是 $b$，这一位可选 $a,c$。
- 每一组的可选字符数量是 $2^{n-1-i}$。
- 同样通过除法和取余来确定选哪一个。

以 $n=3, k=9$ 来举例：

- 总数是 $3 \times 2^{3-1} = 12$。因为 $9 \le 12$，所以存在。
- 第 $1$ 位：每组大小为 $2^{3-1} = 4$。
  - `(9-1) // 4 = 2`。索引 $2$ 对应 $a,b,c$ 中的 **'c'**。
  - 更新 $k = (9-1) \% 4 = 0$。
- 第 $2$ 位：前一位是 $c$，可选位是 $a,b$。每组大小为 $2^{3-2} = 2$。
  - `0 // 2 = 0`。索引 $0$ 对应 $a,b$ 中的 **'a'**。
  - 更新 $k = 0 \% 2 = 0$。
- 第 $3$ 位：前一位是 $a$，可选位是 $b,c$。每组大小为 $2^{3-3} = 1$。
  - `0 // 1 = 0`。索引 $0$ 对应 $b,c$ 中的 **'b'**。
- 所以，最终结果为："cab"。

代码如下，已附加注释：

```Python
# python
class Solution:
    def getHappyString(self, n: int, k: int) -> str:
        # 每个分支有 2^(n-1) 种可能
        group_size = 2 ** (n - 1)
        
        # 总数判断
        if k > 3 * group_size:
            return ""
        
        res = []
        # 确定第一个字符
        choices = ['a', 'b', 'c']
        idx = (k - 1) // group_size
        res.append(choices[idx])
        
        # 在该分支内部的排名
        k = (k - 1) % group_size
        
        # 逐位确定剩下字符
        for i in range(n - 1, 0, -1):
            group_size //= 2  # 下一层的每组大小减半
            
            # 当前可选的两个字符
            last_char = res[-1]
            options = [c for c in ['a', 'b', 'c'] if c != last_char]
            
            # 确定当前位选第几个
            idx = k // group_size
            res.append(options[idx])
            
            # 更新 k
            k %= group_size
         
        return "".join(res)
```

```Java
// java
class Solution {
    public String getHappyString(int n, int k) {
        // 每个分支有 2^(n-1) 种可能
        int groupSize = (int) Math.pow(2, n - 1);

        // 总数判断
        if (k > 3 * groupSize) {
            return "";
        }

        StringBuilder res = new StringBuilder();
        // 确定第一个字符
        char[] choices = {'a', 'b', 'c'};
        int idx = (k - 1) / groupSize;
        res.append(choices[idx]);

        // 在该分支内部的排名
        k = (k - 1) % groupSize;

        // 逐位确定剩下字符
        for (int i = n - 1; i > 0; i--) {
            groupSize /= 2; // 下一层的每组大小减半

            // 当前可选的两个字符
            char lastChar = res.charAt(res.length() - 1);
            char[] options = new char[2];
            int p = 0;
            for (char c : new char[]{'a', 'b', 'c'}) {
                if (c != lastChar) {
                    options[p++] = c;
                }
            }

            // 确定当前位选第几个
            idx = k / groupSize;
            res.append(options[idx]);

            // 更新 k
            k %= groupSize;
        }

        return res.toString();
    }
}
```

- 时间复杂度： $O(n)$，一次循环
- 空间复杂度： $O(n)$，存储结果串
